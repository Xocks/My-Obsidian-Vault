- **一、 搜索与回溯 (DFS/状态空间)**

- **二、 贪心算法 (Greedy) 与双指针**
    
    - **线性单向贪心** $\rightarrow$ **Trigger**：提问“最少操作/最多匹配”且“当前步骤别无选择”。第一反应：单向遍历，绝不回头。
        
    - **有序序列归并/配对** $\rightarrow$ **Trigger**：允许打乱或插入元素求极值。第一反应：必须先 `sort` 排序，然后用双指针“打擂台”，遇到相等字符时 **严格优先原串 (`<`)**。
        
- **三、 高阶数据结构与 STL 运用**
    
    - **双端队列 (Deque)** $\rightarrow$ **Trigger**：输入端频繁在左右切换，或要求滑动窗口。第一反应：弃用 `vector/string` 的头部移位，上 `deque` 或用 `bool is_right` 模拟虚拟光标。
        
    - **差分数组** $\rightarrow$ **Trigger**：允许“相邻区间同时加减”求最终相等/回文。第一反应：彻底抛弃原数组，构造差分数组 `d[i]`，利用正负相消逼近 0。
        
- **四、 极端优化与底层数学**
    
    - **置换环 (图论)** $\rightarrow$ **Trigger**：$1-N$ 乱序排列，允许任意交换求恢复升序最少次数。第一反应：放弃比较排序，套公式 $N - \text{环的个数} K$。
        
    - **周期检测 (Cycle Detection)** $\rightarrow$ **Trigger**：操作次数极度变态（如 $K \le 10^9$）但元素少。第一反应：绝对存在死循环或天花板（最大值），找规律直接对 $N-1$ 取模。

#### 模版 1：二维网格 DFS 回溯终极模版（杀穿一切填数/迷宫）

C++

```
#include <iostream>
#include <vector>
using namespace std;

const int MAX_ROW = 9, MAX_COL = 9, MAX_VAL = 9;
int grid[MAX_ROW][MAX_COL];
bool solved = false;

// 检查在(r, c)填入数字v是否合法（需结合具体题目补充）
bool check_valid(int r, int c, int v) { return true; } 
void mark_used(int r, int c, int v, bool state) { /* 标记/撤销哈希表 */ }

// 函数目的：在二维网格中递归填入合法的数字
// 参数：r (当前行索引), c (当前列索引)
// 返回值：void。状态通过全局变量 grid 和 solved 传递，一旦 solved 变 true，所有分支迅速截断。
void dfs(int r, int c) {
    if (solved) return; // 截断：其他树枝找到答案，立即停止无用功

    if (c == MAX_COL) { // 换行机制
        r++; 
        c = 0; 
    }

    if (r == MAX_ROW) { // 终点：所有行走完
        solved = true; 
        return; 
    }

    if (grid[r][c] != 0) { // 遇到自带数字的格子，直接跳过
        dfs(r, c + 1); 
        return; // 【易死循环点1】必须 return！否则会继续向下执行覆盖原有数字。
    }

    for (int v = 1; v <= MAX_VAL; ++v) {
        if (check_valid(r, c, v)) { 
            grid[r][c] = v;          
            mark_used(r, c, v, true); // 做选择并标记

            dfs(r, c + 1);           // 推进状态

            if (solved) return;      // 【易错点2】找到答案就不要再撤销了，保留现场。

            grid[r][c] = 0;          
            mark_used(r, c, v, false); // 回溯：撤销选择，为 v+1 腾出干净空间
        }
    }
}
```

#### 模版 2：Hoare 双指针快速排序（底层分治思维）

C++

```
#include <iostream>
#include <algorithm>
using namespace std;

// 函数目的：原地将数组区间 [l, r] 升序排序
// 参数：q (待排序的数组指针/名称), l (左边界索引), r (右边界索引)
void quick_sort(int q[], int l, int r) {
    if (l >= r) return; // 递归基：区间只剩1个或非法，直接退出

    // x是基准值，i和j放在区间两头之外，等待 do-while 的第一步踩进来
    int i = l - 1, j = r + 1, x = q[l + r >> 1]; 
    
    while (i < j) {
        do i++; while (q[i] < x); // 左指针找违规的大数
        do j--; while (q[j] > x); // 右指针找违规的小数
        if (i < j) swap(q[i], q[j]);
    }
    
    // 【易越界点1】递归分界线必须是 j。此时 j 及其左边都 <= x。
    quick_sort(q, l, j);       
    quick_sort(q, j + 1, r);   
}
```

### 模块三：极简解题框架（人类思考模拟）

#### 框架一：双端队列 / 状态机模拟（应对左右端点频繁插入/反转）

- **第一步：初始化光标**：绝不用 `reverse()`。开辟 `deque<char> q;`，设置布尔锁 `bool is_right = true;`。
    
- **第二步：核心流转**：
    
    - 遇到反转指令：`is_right = !is_right;`（$O(1)$ 时间复杂度）。
        
    - 遇到插入指令：根据 `is_right` 决定是 `q.push_back()` 还是 `q.push_front()`。
        
- **第三步：防御与输出**：执行删除前必须 `if(!q.empty())`。最后直接用 `for(char c : q)` 顺序输出，物理头部即为逻辑左侧。
    

#### 框架二：差分数组就地消零（应对区间加减/回文构造）

- **第一步：废弃原数组**：强制使用下标从 1 开始。构造差分数组 `d[i] = 原料[i] - 原料[对称位]`。
    
- **第二步：同号对消**：遍历 `d[i]`，只要相邻元素同号（同正或同负），提取最小绝对值 `mn`。正数同时减，负数同时加。
    
- **第三步：残局结算**：经过白嫖对消后，剩下无法结对的元素，直接将其绝对值累加到答案中。

### 模块四：本周高价值 BUG 库与降维打击（提纯易错点）

#### 1. 数组访问与函数调用的概念混淆 (P8597 翻硬币)

C++

```
// 你的原代码
if(arr[i]!=brr[i]) {
    arr[i] = fan[arr[i]]; // ❌ 错将函数当数组。fan 是函数工厂，不是储物柜。
    arr[i+1] = fan[arr[i+1]]; 
    res++;
}
```

- **纠错**：改为 `fan(arr[i])`。必须用圆括号传参。
    
- 🚀 **降维打击**：**空间压缩至 $O(1)$**。完全不需要额外开 `arr` 和 `brr`。直接在原字符串 `s` 上操作：`if(s[i] != target[i]) { s[i] ^= 1; s[i+1] ^= 1; }` （利用异或直接翻转字符，连函数都不需要调用）。
    

#### 2. 无符号整数下溢出死循环 (for 循环条件雷区)

C++

```
// 你的原代码
for(int i = 0; i < str.size() - 2; ++i) // ❌ size_t 下溢出陷阱
{
    if(tes(i)) { i += 2; res++; }
}
```

- **纠错**：`str.size()` 返回的是无符号整数。如果 `size < 2`，减去 2 会变成 $1.8 \times 10^{19}$ 的极大正数，导致越界死循环。改为：`i + 2 < str.size()`。
    

#### 3. while 循环分支缺失导致 TLE (牛客 C 小苯观景)

C++

```
// 你的原代码
while(r<n) {
    int x = vec[r] - vec[l];
    if(x>=tes) { tes++; res++; l=r; r++; }
    // ❌ 致命缺失：没有 else。当 x < tes 时，r 和 l 都不动，陷入无限死循环 TLE。
}
```

- **纠错**：加上 `else { r++; }`，让右指针继续探索。
    

#### 4. 判断 2 的幂次的递归灾难 (试题 D 卡牌合并)

C++

```
// 你的原代码
int coss (long long num) {
    // ...
    else {
        return cos(num/2); // ❌ 命名冲突灾难。误调了 <cmath> 里的三角函数余弦值！
    }
}
```

- 🚀 **降维打击**：**位运算 $O(1)$ 秒杀**。任何 2 的次幂在二进制下只有一位是 1。直接写：`if (num > 0 && (num & (num - 1)) == 0)`。不仅规避了函数名冲突，连递归的栈空间和 $O(\log N)$ 的时间都省了。

**【自测答案区】**

- **A1**：会触发“未定义行为 (UB)”甚至段错误 (RE) 崩溃。因为对空队列调用 `.front()` 是违法的。必须加上防御：`if (!q.empty() && s[i] == q.front())`。
    
- **A2**：隐患是“短视”。如果原串后续藏着更小的字典序，提前插入相等的字符会阻塞原串。例如：原串 `str = "bba"`, 插入串 `ins = "b"`。如果用 `<=`，会先输出插入的 `b`，再输出原串的 `bba`，结果是 `bbba`。如果严格用 `<`，会先让原串走完前面的 `b`，等到后面的潜力展现，最终输出是最优。
    
- **A3**：死公式：`当前列 + 当前行 * 每行长度`，即 `c + r * len`。
    
- **A4**：**绝对不会停下！** 因为 `size_t` 是无符号类型，它永远不可能小于 0。当 `i` 从 0 减 1 时，会下溢出变成其类型支持的最大正数，导致无限循环和越界崩溃。
    
- **A5**：结果是 `true`（因为 $0 \ \& \ -1 = 0$）。但这在逻辑上是错误的，因为 $0$ 不是 $2$ 的任何次幂。防线必须加上 `n > 0` 的判断：`n > 0 && (n & (n-1)) == 0`。
