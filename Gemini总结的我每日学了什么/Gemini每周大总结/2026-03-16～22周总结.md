### 模块二：极简解题框架（傻瓜式操作指南）

针对本周最拉分的三个核心题型，请将以下“三步走”刻进潜意识：

#### 框架 A：资源分配/组合型 DFS

- **第一步：明确参数与底线（初始化）**。问自己：递归到了第几个坑位？手里剩多少资源？去重底线（`last`）是什么？在函数最开头写下防爆剪枝（`if (资源超标) return;`）。
    
- **第二步：明确终点（防越界）**。门卫拦截：必须等到**坑位全部填满**（如 `if (p == n)`），再去结算目标是否达成。绝不能因为资源发完了就提前结束。
    
- **第三步：横向枚举与回溯（核心循环）**。用 `for` 循环枚举当前坑位能拿多少。起点受制于去重规则，终点受制于剩余资源。带着扣除后的资源进入下一层，出来时撤销标记（如果用了全局状态）。
    

#### 框架 B：多源 BFS 水波纹蔓延

- **第一步：全局初始化**。把整个地图距离设为 -1。找到**所有**起点，距离赋为 0，并全部 `push` 进队列。
    
- **第二步：提取层级（核心循环）**。`while (!q.empty())`，提取队头。如果需要按层计时，提前锁死当前层的 `current_size = q.size()`。
    
- **第三步：四向扩散与越界审查**。`for` 循环走上下左右。只要不越界、不是障碍，且**距离还是 -1**，立刻更新距离（当前距离+1）并压入队尾。
    

#### 框架 C：二分答案 + 贪心校验

- **第一步：确定边界与存档**。`left = 1`，`right = 极限值`，初始化 `ans = 0` 用于存档合法结果。
    
- **第二步：对半切与校验**。`while (left <= right)`，切出 `mid = left + (right - left) / 2`，把 `mid` 当作硬性要求，传给 `check(mid)` 函数遍历一遍验证。
    
- **第三步：收缩与提取**。如果 `check` 合法，`ans = mid` 存档，去尝试更贪心的边界（`left = mid + 1` 或反之）；如果不合法，缩小预期（`right = mid - 1`）。最后输出 `ans`。

### 模块三：全量知识点与核心思路精讲

- **DFS 的底线思维**：在分配问题中，“动作（发一颗糖）”不是深度，“阶段（第几个人）”才是深度。DFS 本质是让计算机代替你画出一棵极其庞大的决策树，走到死胡同就退回来。
    
- **BFS 的物理直觉**：为什么网格连通块和最短路要用 BFS？因为 DFS 是一条路走到黑，极易导致系统栈溢出（爆栈）。BFS 借用队列的“先进先出”特性，完美模拟了真实世界中水波纹按“层”扩散的物理现象。谁先摸到终点，谁就绝对是最短路径。
    
- **DP 与 DFS 的降维对抗**：DFS 关心的是“我具体怎么走”，而 DP 关心的是“我这个状态能由哪些已知状态推导过来”。在《完美方阵》中，DP 将向外无脑画圈的 $O(N^4)$ 动作，转换为了优雅地回头看左、上、左上方三个邻居的 $O(N^2)$ 动作。
    
- **二分答案的逆向工程**：在 $5 \times 10^4$ 个石头里挑出 $M$ 个拔掉，组合数是天文数字。二分答案彻底逆转了思维：我不去想要拔哪几块，我直接“蒙”一个最短跳跃距离，然后用一遍 $O(N)$ 的贪心去扫，看这个距离是否要求我拔掉超标的石头。

### 模块四：源码级纠错与降维打击（重点！）

#### 1. 致命后缀陷阱 (原味纠错：The Rock Game)

C++

```
void dfs (int st,int num) {
    if(arr.size()>n)return; 
    // ...
    arr.push_back(0);
    // ❌ 错误点：st++ 会先把当前旧的 st 传给下一层，然后再自增。
    // 这导致下一层收到的永远是没加过的 st，引发无限递归爆栈。
    // 💡 纠正：必须写成 dfs(st + 1, num);
    dfs(st++,num); 
    arr.pop_back();
}
```

### 模块五：核心代码模版与保姆级拆解

为你提取本周最重要的跨阶级模版：**二分答案 (以跳石头为例)**。这套代码架构可以秒杀所有“最大化最小值”的题目。

C++

```
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 50010;
int stone_dist[MAXN]; 
int total_len, stone_num, max_remove;

// 🎯 参数级解析：guess_dist 是我们正在试图挑战的"最短跳跃距离"
// 返回值：如果在该距离下，必须拔掉的石头数量在允许范围内，返回 true
bool check(int guess_dist) {
    int remove_cnt = 0;   
    int current_pos = 0;  // 永远记录你当前脚踩在哪

    // 避坑指南：必须遍历到 stone_num + 1，连同最后一步跳向终点的距离也要测！
    for (int i = 1; i <= stone_num + 1; ++i) {
        if (stone_dist[i] - current_pos < guess_dist) {
            remove_cnt++; // 距离不够，这块石头太碍事，必须拔掉
        } else {
            current_pos = stone_dist[i]; // 距离够了，跳过来，更新脚位
        }
    }
    return remove_cnt <= max_remove; 
}

int main() {
    ios::sync_with_stdio(false); cin.tie(0);
    cin >> total_len >> stone_num >> max_remove;
    for (int i = 1; i <= stone_num; ++i) {
        cin >> stone_dist[i]; 
    }
    stone_dist[stone_num + 1] = total_len; // 极易错的边界：终点必须加进数组

    int left = 1;
    int right = total_len;
    int ans = 0; // 永远用 ans 来存档

    while (left <= right) {
        // 避坑指南：用减法代替加法除二，防止数据过大导致 int 溢出
        int mid = left + (right - left) / 2; 
        
        if (check(mid)) {
            ans = mid;       // 满足条件，先把当前成果锁进保险箱
            left = mid + 1;  // 野心变大，尝试要求更远的跳跃距离
        } else {
            right = mid - 1; // 步子太大扯着蛋了，必须降低要求
        }
    }

    cout << ans << "\n";
    return 0;
}
```

**【答案核对区】**

1. **答案**：`st++` 是后缀运算，它会先提取当前 `st` 的值硬塞给下一层函数，然后再给本层的 `st` 加 1。这导致无限将相同的值传递下去，死循环爆栈。`st+1` 才是真正计算好下一步的状态再传递。
    
2. **答案**：必报段错误崩溃。$x$ 代表行，最大值可到 $n-1$，你却用较小的列数 $m$ 去限制它，导致合法行被错误拦截；而代表列的 $y$ 会用较大的 $n$ 去限制，导致它能轻易突破实际列边界 $m$，访问到非法内存。
    
3. **答案**：C++ 底层排序算法要求“严格弱序”。如果两个元素相等时你返回了 `true`，迭代器会无法收敛，直接越界导致程序崩溃（Segmentation Fault）。
    
4. **答案**：内存与时间双重爆炸 (MLE/TLE)。两个相邻的空地会互相视对方为“没走过”，然后互相疯狂把对方推进队列，导致队列无限膨胀永远无法清空。
    
5. **答案**：浮点数在计算机底层是以近似值存储的，存在精度丢失。你以为存入的是 $0.1$，底层可能是 $0.100000001$。用它作为 Hash 的 Key，极大概率会匹配失败，产生找不到数据或意外覆盖的幽灵 Bug。
