### 1. 状态空间探索（穷举与回溯）

这是本周最核心的骨架，所有题目的起点都在于“如何不重不漏地遍历所有可能性”。

- **指数型枚举（选或不选）：** 关注物品本身，利用平行分支推进。
    
- **排列型枚举（位置找人）：** 关注坑位，利用 `for` 循环横向扫视配合状态数组。
    
- **全排列等价（`next_permutation`）：** 当不需要中途剪枝且只需顺序组合时，跳过手写 DFS，直接利用内置函数降维打击。
    
- **内在联系：** 它们本质上都是在遍历一棵“决策树”。DFS 是一种**通用但昂贵**的武器，而剪枝（预判死胡同提前 `return`）则是让这把武器真正能上赛场的磨刀石。
    

### 2. 等价转换与数学思维（降维打击）

当状态空间大到 DFS 甚至剪枝都无法拯救时，你开始学习寻找“破局点”。

- **周期模拟转数学取余：** 漫长的回合制对战，利用最小公倍数 (LCM) 瞬间跨越时间轴。
    
- **物理过程转极值推导：** 不去一步步模拟灌木的生长，而是直接揪出“距离端点最远”的极端情况，利用公式 $\max(L, R) \times 2$ 一招毙命。
    
- **复杂平分转贪心目标：** 将动态的“加减操作”转化为对静态不变量（向上取整的平均值）的差值填补。
    

### 3. C++ 底层机制与状态流转

- **缓冲区清洗：** `cin` 与 `getline` 的交锋。
    
- **状态机（滚动变量）：** 抛弃十万级大数组，仅用 `curr_ms` 和 `last_ms` 两个变量交接班，将空间压缩至极。

## 二、 高频盲区与核心避坑指南

纵观这一周的试错，你的宏观思路往往精准，但极其容易在 C++ 的底层语法边界和特殊测试用例上反复流血。

### 核心思维盲区

- **未滤净的“脏数据”信任：** 习惯性地信任未初始化的局部变量或越界的数组下标（如 `n=1` 时的 `arr[1]`），导致程序逻辑被内存垃圾带偏。
    
- **负数域的“理所当然”：** 在进行除法取整或 ASCII 码计算时，没有考虑 C++ 底层对负数的向零取整机制，以及字符到数字的硬编码转换鸿沟。
    

### 3 条必须默念的“避坑铁律”

1. **“万物归零法则”：** 凡遇矩阵坐标、周期循环、数组取模，第一反应必须是 `m--; n--;` 降维到 0 基准！算完再加回去，彻底杜绝跳行与余数归零的错乱。
    
2. **“防爆防漏法则”：** 超过 1000 的大数组**必须**开在全局区（自动清零且防爆栈）；局部变量声明时**当场**赋值 `= 0`；只要 `if` 后面的动作超过一个，哪怕再短也必须加大括号 `{}`。
    
3. **“先决动作法则”：** 调用 `next_permutation` 前，**必须**先 `sort`；只要前面用了 `cin >>` 且后面要用 `getline`，中间**必须**用 `cin.ignore` 清理战场。

## 三、 核心模版库与更优解法（降维打击）

大浪淘沙，以下是为你提取的最具实战价值的核心 C++ 模版。

### 模版 1：DFS 组合搜索终极框架（带可行性剪枝）

这是你本周接触最深的模型，适用于需要中途打断、状态复杂的组合问题。

C++

```
#include <iostream>
#include <vector>
using namespace std;

vector<int> path; // 全局变量：存放当前的单条路径

// 函数目的：递归探索所有可能的组合
// st: 传递当前所在的层级/步骤（比如正在决定第几个坑位）
// cur: 传递当前的累积状态（比如当前配料的总重量）
// target: 传递目标值（比如披萨要求的总重量）
void dfs(int st, int cur, int target) {
    // 1. 终极剪枝：提前排除绝对不可能的情况
    // 比如：当前重量加上剩余全选最轻的仍然大于目标，直接回头
    if (cur > target) return; 
    
    // 2. 终止边界条件
    if (st > 10) { 
        // 到达叶子节点，验证最终状态
        if (cur == target) {
            // 收集或打印 path
        }
        return; // 【错误点注】：这里的 return 必须在边界判断的 if 内部！
    }
    
    // 3. 遍历当前层级的所有选择
    for (int i = 1; i <= 3; ++i) { // 假设每种配料有 1~3 克可选
        path.push_back(i);            // 动作 1：做出选择，记录轨迹
        dfs(st + 1, cur + i, target); // 动作 2：带着新状态 (cur + i) 进入下一层
        path.pop_back();              // 动作 3：回溯核心！撤销选择，恢复现场给下一个 for 循环使用
    }
}
```

**更优解法对比：** 纯暴力的 DFS 时间复杂度是 $O(3^{10})$，加入 `if (cur > target)` 的可行性剪枝后，无效的庞大分支被提前砍掉，时间复杂度呈断崖式下降。

### 模版 2：安全向上取整（彻底告别负数与除法坑）

在处理贪心和数学平分问题时，此模版能救命。

C++

```
typedef long long ll;

// 函数目的：计算 a / b 的严格向上取整
// a: 传递被除数（总和或差值，可能为负）
// b: 传递除数（通常为正数，如数组长度 n）
ll ceil_div(ll a, ll b) {
    // 【错误点注】：切勿使用 (a+b-1)/b 通吃所有情况。
    // 当 a 为负数时，C++ 的整数除法自带向 0 取整机制，逻辑会崩溃。
    if (a >= 0) {
        return (a + b - 1) / b; // 正数域的标准向上取整
    } else {
        return a / b;           // 负数域直接除，C++ 会自动向 0（即向上）取整
    }
}
```

### 模版 3：无死角格式化字符串提取

用于处理含有字母、符号、数字混合的恶心输入流。

C++

```
#include <iostream>
#include <string>
#include <cstdio> // sscanf 所在库
using namespace std;

void parse_time() {
    string line;
    getline(cin, line);

    // 必须给可选参数赋初值，防止 sscanf 没匹配到时产生垃圾脏数据
    int h1, m1, s1, h2, m2, s2, days = 0; 
    
    // sscanf 直接按照模版串从 line 中抠出对应的整数
    // %d 对应整型，固定字符（如 : 和 +）作为匹配锚点
    sscanf(line.c_str(), "%d:%d:%d %d:%d:%d (+%d)", &h1, &m1, &s1, &h2, &m2, &s2, &days);
}
```

**更优解法对比：** 相比于手工用 `(tim[0] - '0') * 10` 逐个字符切片累加，`sscanf` 将 20 行极易越界的代码缩减至 1 行，彻底规避了 ASCII 码偏移量算错的雷区。
