### 模块一：今日知识清单与特定触发机制

- **深度优先搜索 (DFS) 与回溯**
    
    - **Trigger (触发条件)：** 题目出现“求所有方案”、“求所有组合/拆分方式”，且数据范围极小（通常 $N \le 20$）。看到“具体列出所有可能”，第一反应必须是 DFS。
        
- **占位符与去重原则 (人为设定单调性)**
    
    - **Trigger (触发条件)：** 题目明确表示“不计顺序（如 1,1,5 和 5,1,1 算一种）”。第一反应是必须传递 `last` 变量，强制后续选择 $\ge$ (或 $\le$) 当前选择，且**必须走完全部流程（填满所有占位符/空盘子）才能结算**，绝不能提前 `return`。
        
- **前缀和数组 (Prefix Sum)**
    
    - **Trigger (触发条件)：** 题目涉及“多次查询区间 $[L, R]$ 的总和”。第一反应是预处理 `sum[i] = sum[i-1] + arr[i]`，查询时直接 $O(1)$ 计算 `sum[R] - sum[L-1]`。
        
- **差分数组 (Difference Array)**
    
    - **Trigger (触发条件)：** 题目出现“频繁对某一时间段/区间 $[L, R]$ 整体加减（如某时间段有学生占用游乐场）”，最后求某个时刻的极值。第一反应是 `diff[L]++`, `diff[R]--` （如果是左闭右开），最后做一次前缀和还原真实数据。
        
- **多源广度优先搜索 (Multi-source BFS)**
    
    - **Trigger (触发条件)：** 题目出现“像水波纹一样同时向外扩散”、“多起点同时感染”、“求最短时间/步数”，且处于网格或图中。第一反应必须是**队列 (Queue)**，绝不能用嵌套 `for` 循环暴力模拟。

### 模块二：傻瓜式解题框架（思路提取）

针对今天的核心难点**【组合/拆分方案类 DFS】**，你的脑海中必须刻下这个 3 步思考框架：

1. **第一步：确定状态参数（你在哪，你要去哪，你的底线是什么）**
    
    - 传递 `p` 或 `index`：表示当前在做第几次决策（处理第几个盘子）。
        
    - 传递 `sum`：记录目前的累加进度。
        
    - 传递 `last`：记录去重底线（上一次选了什么），保证单调性。
        
2. **第二步：明确终点与防爆（边界处理）**
    
    - **防爆（剪枝）：** 在函数最开头，判断客观物理限制（如 `if (sum > target) return;` 提前掐断死路）。
        
    - **终点（结算）：** 必须等到决策次数/占位符耗尽（如 `if (p == n)`），再去判断目标是否达成（`if (sum == target)`）。除非你采用了将“空占位符 0”全部垫底的降序策略，否则严禁中途结算。
        
3. **第三步：横向枚举（For 循环的核心逻辑）**
    
    - **下界（起点）：** 永远受限于你的规矩（为了不重复，起点通常是 `last`）。
        
    - **上界（终点）：** 永远受限于现实（你手里还剩多少资源，如 `target - sum`。注意判断是否允许一次性“梭哈”，如果允许，必须带 `=` 号）。
        
    - **行动与撤销：** 执行 `dfs(next_state)`。如果使用了全局数组记录路径，必须在 DFS 之后紧跟 `.pop_back()` 恢复现场。

### 模块三：我的源码级纠错与降维打击

#### 1. 原味纠错：放苹果 (DFS 边界与逻辑错误)

C++

```
void dfs(int last,int sum,int p)
{
    // 你的原代码：if(sum>num)return; if(p==n){ ... }
    
    // 错误点：上界缺失等于号。
    // 纠正原因：当你手里还剩 (num-sum) 个苹果时，题目允许你把剩下的全部放进当前的盘子里（后面的盘子全空着，放0）。
    // 如果没有 '='，你永远无法把苹果全部分配完，导致最终 sum 永远到达不了 num，结果输出必然为 0。
    for(int i = last;i<num-sum;++i) // 必须改为：i <= num - sum
    {
        dfs(i,sum+i,p+1);
    }
    return;
}
```

#### 🚀 降维打击（替代算法）

**替代对象：** 《自然数拆分》与《放苹果》（当仅仅只问“有多少种方案”，且 $N, M$ 范围高达 $1000$ 时）。

**你的原算法：** 深度优先搜索 (DFS)。时空复杂度呈指数级爆炸 $O(\text{方案数})$。

**降维算法：动态规划 (DP) - 完全背包模型衍生。**

**为什么更优：** 人类在脑海中模拟是一步步放，但 DP 直接从宏观数学分类入手。时间复杂度降至 $O(M \times N)$。它不关心具体是怎么放的，只关心状态的转移：`当前方案数 = (至少一个空盘子的方案数) + (每个盘子至少放一个的方案数)`。

**核心转移方程：** $dp[i][j] = dp[i][j-1] + dp[i-j][j]$ (用我的变量名：$i$ 为苹果，$j$ 为盘子)。

**替代对象：** 《腐烂的苹果》（水波纹扩散问题）。

**你的原算法：** 全局扫描暴力模拟。时间复杂度高达 $O((X \times Y)^2)$ 且极易写崩。

**降维算法：多源 BFS（广度优先搜索）。**

**为什么更优：** 时间复杂度极低 $O(X \times Y)$。利用队列“先进先出”的物理特性，完美模拟“同时向外发散一层”的物理直觉。你不需要全局找是谁烂了，烂苹果自己排着队去感染别人。

### 模块四：核心代码模版与保姆级拆解

这里为你提供必须刻在 DNA 里的**多源 BFS 标准模版**（针对腐烂的苹果类层级扩散问题）：

C++

```
#include <bits/stdc++.h>
using namespace std;

// 1. 方向数组 (技巧)：用来快速生成上下左右的坐标，避免写 4 个冗长的 if
int dx[4] = {-1, 1, 0, 0};
int dy[4] = {0, 0, -1, 1};

int main() {
    int x, y;
    // ... 读取网格数据 ...
    
    queue<pair<int, int>> q; // 核心数据结构：队列，专门存烂苹果的坐标
    int fresh = 0;           // 记录当前新鲜苹果数量
    
    // 2. 初始化：把所有初始烂苹果丢进队列 (多源的精髓)
    for(int i = 0; i < x; ++i) {
        for(int j = 0; j < y; ++j) {
            if(grid[i][j] == 2) q.push({i, j});
            if(grid[i][j] == 1) fresh++;
        }
    }
    
    if(fresh == 0) { cout << 0 << '\n'; return 0; }
    int minutes = 0;
    
    // 3. 开始 BFS 水波纹扩散
    while(!q.empty()) {
        int current_size = q.size(); // 极其关键：锁死当前这一分钟要处理的数量
        bool infected = false;       // 标记这一分钟有没有干活
        
        // 只处理当前这一层的烂苹果
        for(int i = 0; i < current_size; ++i) {
            auto curr = q.front();
            q.pop();
            
            // 尝试向四个方向感染
            for(int d = 0; d < 4; ++d) {
                int nx = curr.first + dx[d];
                int ny = curr.second + dy[d];
                
                // 边界与合法性审查
                if(nx >= 0 && nx < x && ny >= 0 && ny < y && grid[nx][ny] == 1) {
                    grid[nx][ny] = 2; // 感染！
                    fresh--;
                    q.push({nx, ny}); // 新烂的苹果排到队尾，下一分钟再去害人
                    infected = true;
                }
            }
        }
        if(infected) minutes++; // 这一层（这一分钟）全都扩散完了，时间才 +1
    }
    
    if(fresh > 0) cout << -1 << '\n';
    else cout << minutes << '\n';
    return 0;
}
```

**参数与细节透视：**

- `queue<pair<int, int>> q`：队列。`push()` 放队尾，`front()` 看队头，`pop()` 踢出队头。存放的永远是**下一秒即将向外扩散的感染源**。
    
- `current_size = q.size()`：**层级控制的核心。** 如果没有它，队列边出边进，你根本不知道哪几个苹果是属于“同一分钟”烂掉的，也就无法正确记录时间。
    
- `infected` 布尔值：如果最后一次出队的烂苹果周围全是空的或已经是烂的，它没有感染任何人，那么这“一分钟”是无效的，不该被加到总时长里。

### 模块五：今日高频盲区与灵魂自测

**高频盲区 1：越界即毁灭。** 在字符串遍历 `str[i+j]`、网格探测 `grid[nx][ny]` 中，不加限制地加上一个偏移量，是最容易导致段错误 (Segmentation Fault) 的死穴。

**高频盲区 2：强制单调性与提前结算的冲突。** 为了去重设定了 `last`，就把“占位符 0”逼到了特定位置。在 0 没有完全落位前，绝对不能根据 `sum == target` 提前 `return`。

#### 灵魂自测 (请遮挡答案食用)

**Q1：在多源 BFS 模版中，如果不写 `int current_size = q.size();` 以及内层的 `for` 循环，直接 `while(!q.empty())` 弹出一个处理一个，然后在末尾 `minutes++`，会导致什么灾难性后果？**

> **答案：** 时间计算会彻底崩溃。原本“同一分钟内”同时向外扩散的 4 个苹果，会被拆分成先后发生的 4 件事。导致本来只过去 1 分钟，你的代码却给 `minutes` 加了 4。失去了“层级（水波纹）”同步推进的物理意义。

**Q2：在《放苹果》的 DFS 代码中，假设手里剩下 5 个苹果，当前是最后一个盘子，如果我把循环条件写成了 `for(int i = last; i < num - sum; ++i)` (少了等于号)，在逻辑层面我实际向计算机下达了什么荒谬的指令？**

> **答案：** 你下达的指令是：“即使你手里剩 5 个苹果，并且这是最后一个盘子，你最多也只准在这个盘子里放 4 个。” 这直接导致剩下的 1 个苹果凭空消失，永远无法满足 `sum == target` 的通关条件。

**Q3：在处理“找最便宜且附带名字的模型”这类问题时，为什么直接使用 `map<价格, 名字>` 是一种极其危险甚至可以说是完全错误的做法？（说出两点原因）**

> **答案：** > 1. **键值覆盖规则冲突：** 题目明确要求“如果价格相同，输出最早输入的”。但 Map 是一种哈希映射，当你插入第二个相同价格的模型时，它的名字会直接覆盖掉第一个模型，完美违背了题目要求。
> 
> 2. **浮点数作为 Key 的大忌：** 浮点数 (float/double) 在计算机底层存储存在精度丢失（如 0.1 可能存为 0.100000001）。用浮点数作为 Map 的 Key 去索引，极大概率会发生“明明价格一样，但 Map 认为它们不同”的幽灵 Bug。找最值，老老实实打擂台最靠谱。
