### 模块二：考场条件反射（Trigger）

- **触发点 1：数据范围 $A_i \le 10^5$ 且涉及乘法/平方**
    
    - **条件反射**：无脑开 `long long`！不仅存储数组要开，所有参与接收加法累加、乘法结果的**中间变量**（如求和变量 `sum`、差值变量 `cor`）必须同步改为 `long long`。
        
- **触发点 2：看到公式 $\sum |A_{i+1} - A_i|$ 或相邻两项差值求和**
    
    - **条件反射**：绝对不要去写滑动窗口或者用新数组存差值！立刻想到**裂项相消**。连续 $M$ 项的内部相邻差值相加，中间项必定全部抵消，永远只等于 **“最后一项减去第一项”**。
        
- **触发点 3：已知二维坐标 $(x, y)$ 求圆心角/旋转角度**
    
    - **条件反射**：直接调用 `atan2(y, x)`，它自动处理坐标系象限和 $x=0$ 的边界，返回最干净的弧度值。结合弧长公式 $s = r\theta$ 即可求解距离。

### 模块三：傻瓜式解题框架（模拟人类思考）

_针对核心题型：连续区间相邻差（平方差）极小值问题（画展布置）_

- **第一步：如何初始化/定义状态？**
    
    读入所有画作价值后，**必须先升序排序**。因为只有把数放在一条轴上连续取，首尾的跨度才可能最小。接着，定义一个初始值为极大值的变量（如 `9e18`），用来随时记录打擂台碰到的最小代价。
    
- **第二步：核心逻辑/循环的推导方向是什么？**
    
    脑子里抛弃对每一项依次相减的执念。我们只需要用一个定长为 $M$ 的无形框子，从排序后的数组开头滑到结尾。因为裂项相消的性质，框子里这 $M$ 个画作的价值差总和，永远只看框子的“右边界元素的平方”减去“左边界元素的平方”。
    
- **第三步：如何处理输出和边界？**
    
    循环的起点是 $0$，终点是 $N - M$（保证框子右侧不越界）。在循环内直接算一次首尾平方差，和记录最小代价的变量比大小，存下更小的一个。循环结束，直接输出。

### 模块四：逻辑切片与源码纠错（核心精华）

#### 1. 灵魂 5 行（画展布置最优解）

C++

```
sort(hua.begin(), hua.end()); // 为什么这么写：将无序变有序，保证连续取值的首尾差值必定是全场最小的可能。
for (int i = 0; i <= n - m; ++i) { // 为什么这么写：定长为 m 的区间，i 是左端点，i+m-1 是右端点。
    long long current_L = hua[i + m - 1] * hua[i + m - 1] - hua[i] * hua[i]; // 为什么这么写：裂项相消的精髓，中间所有元素的平差方相加全抵消了，只剩尾的平方减头的平方。
    if (current_L < min_L) min_L = current_L; // 为什么这么写：打擂台，留存历史最小代价。
}
```

#### 3. 降维打击

- **替代算法**：将你的“先存入差值数组 `l`，再用滑动窗口双指针累加计算”的方法，替换为“**裂项相消直接算首尾**”。
    
- **优势**：直接干掉了 `l` 数组和所有麻烦的左右指针加减逻辑，把时空复杂度里的空间开销从 $O(N)$ 降到 $O(1)$，代码从近 50 行缩减至极简的遍历。

### 模块五：抗遗忘灵魂自测（主动回想）

**今日最易踩坑盲区：**

1. **半吊子 Long Long**：以为给数组开了 `vector<long long>` 就万事大吉，忽略了接收运算结果的独立变量也必须开 `long long`。
    
2. **暴力求和强迫症**：看到公式里的 $\Sigma$ 求和符号就忍不住去写 `for` 循环累加，完全忘了观察相邻项是否存在正负抵消的数学漏洞。
    

**自测问答题（请先遮挡下半部分作答）：**

- **问题 1**：在 `vector<long long> hua` 的情况下，为什么写 `int cor = hua[1]*hua[1] - hua[0]*hua[0]` 会导致致命错误？具体发生了什么物理过程？
    
- **问题 2**：如果题目要求在数组中挑 $M$ 个数使得相邻差值之和最小，为什么第一步必须是排序？如果不排序直接挑会有什么后果？
    
- **问题 3**：在计算几何题中求极角，为什么必须优先用 `atan2(y, x)` 而不是用高中的 `atan(y/x)`？
    

**------ 答案区 ------**

- **答案 1**：虽然右侧的计算由于数组是 `long long` 会在 64 位寄存器下安全完成，但当把结果（比如 $10^{10}$）赋值给左侧的 32 位 `int cor` 时，多出的高位 bit 会被直接“一刀切”截断。这导致 `cor` 不仅数值面目全非，符号位也可能翻转变成负数，彻底毁掉后续判断。
    
- **答案 2**：因为只有将数组放在一维坐标轴上升序排列，依次选取的点组成的“折线”才不会“走回头路”。不排序的话，选取的相邻数字可能一会儿大一会儿小，差值的绝对值累加起来会包含大量冗余的路径代价，绝对无法得到最小值。
    
- **答案 3**：`atan(y/x)` 存在两个死穴：第一是当 $x=0$ 时会发生除零异常（程序直接崩溃）；第二是它无法区分第一象限和第三象限（例如 $y=-1, x=-1$ 和 $y=1, x=1$ 算出的比值都是 1）。而 `atan2(y, x)` 内部自带象限判定和边界处理，可以直接返回准确的 $[-\pi, \pi]$ 范围内的真实弧度。
