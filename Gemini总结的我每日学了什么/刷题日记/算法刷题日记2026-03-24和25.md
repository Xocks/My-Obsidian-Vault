## 模块二：傻瓜式解题框架（思路提取）

针对今天最容易卡壳的**“字符串周期分段对齐”**题型，提取极其死板的解题步骤框架：

**第一步：边界判定与初始化**

- 计算总长度 `so = str.size()`。
    
- 做数学拦截：如果 `so % k != 0`，说明根本无法整除，直接输出 `-1` 并 `return 0`。
    
- 计算出每一小段的真实物理长度：`int len = so / k;`。
    

**第二步：外层循环控“列”，建立哈希隔离**

- 写外层循环遍历每一列：`for (int i = 0; i < len; ++i)`。
    
- **极其重要：** 在外层循环刚进去的第一句话，立刻声明一个干净的空桶 `vector<int> count(26, 0);` 和最大频次记录器 `max_freq = 0;`。保证列与列之间数据绝对隔离。
    

**第三步：内层循环控“行”，坐标映射与统计**

- 写内层循环遍历每一段（行）：`for (int j = 0; j < k; ++j)`。
    
- 套用一维转二维死公式算出真实下标：`int num = i + j * len;`。
    
- 扔进桶里：`int ind = str[num] - 'a'; count[ind]++;`，并顺手更新 `max_freq`。
    

**第四步：局部贪心结算**

- 内层循环结束后，说明这一列查完了。将这列需要修改的代价累加到总答案：`m += (k - max_freq);`。

### 1. 语法越界与逻辑错乱 (B4482)

你的原逻辑跑偏在多维数组的下标计算和 DFS 步进方向的不一致。

C++

```

{
    // ... 前面的换行和终点逻辑没问题 ...

    for(int v = 1;v<=4;++v)
    {
        // ❌ 错误点 1：数组运算优先级错误。

        // 纠正：除法必须在括号内计算 -> block_used[r/2][c/2][v]
        if(!row_used[r][v]&&!col_used[c][v]&&!block_used[r/2][c/2][v])
        {
            grid[r][c] = v;
            // ... 打标记 ...
            
            // ❌ 错误点 2：DFS 步进方向与初始设定矛盾。

            // 但这里你写成了 dfs(c, r+1)，强行让机器人向下跳行，导致漏掉当前行的剩余格子。
            // 纠正：保持向右走 -> dfs(c+1, r);
            dfs(c,r+1);

### ⚔️ 降维打击：双指针替换队列 (针对 P8697)

- **你的原思路：** 开一个 `queue` 把短字符串塞进去，空间复杂度是 $O(|T|)$。
    
- **替代算法（双指针法）：** 直接用一个整数变量 `j` 当作指向短串的“游标”。
    
- **为什么更优：** 空间复杂度瞬间从 $O(|T|)$ 降维到**绝对的 $O(1)$**。无需任何额外内存分配，且免去了 `queue` 底层封装的常数时间开销。代码更短，跑得更快，且不用担心 `.front()` 带来的空指针异常。
    

C++

```
    int match_ptr = 0; // 替代队列的游标
    for(int i = 0; i < s.size(); ++i) {
        if(match_ptr < t.size() && s[i] == t[match_ptr]) {
            match_ptr++; 
        }
    }
```

### 3. 贪心基准选错 (P8739)

你找到了循环比对的框架，但“第一印象即真理”的逻辑毁了全局最优。

C++

```

    {

## 模块四：核心代码模版与保姆级拆解

今天最核心、最值得背诵的通用模版是**二维网格的 DFS 回溯模版**。它能秒杀 90% 的填数、走迷宫问题。

### 完整代码

C++

```
// 模版：二维网格填数型 DFS
void dfs(int r, int c) {
    // 1. 找到答案后的全局截断
    if (solved) return; 

    // 2. 坐标换行处理
    if (c == MAX_COL) { 
        r++; 
        c = 0; 
    }

    // 3. 终点判定 (所有行都走完了)
    if (r == MAX_ROW) { 
        solved = true; 
        print_grid(); 
        return; 
    }

    // 4. 跳过已固定/不需要填的障碍物
    if (grid[r][c] != 0) { 
        dfs(r, c + 1); 
        return; // 极易漏掉的 return
    }

    // 5. 核心：做选择 -> 递归 -> 撤销选择 (回溯)
    for (int v = 1; v <= MAX_VAL; ++v) {
        if (check_valid(r, c, v)) { 
            grid[r][c] = v;          // 做选择
            mark_used(r, c, v, true); // 打上限制标记

            dfs(r, c + 1);           // 带着当前状态走向下一步

            if (solved) return;      // 成功了就不要再回溯破坏现场了

            grid[r][c] = 0;           // 撤销选择
            mark_used(r, c, v, false); // 擦除限制标记，给下一个 v 腾地方
        }
    }
}
```

### 逐行剖析与参数透视

- **参数传递：** `r` 代表当前所在的行索引，`c` 代表列索引。它们共同确定了“机器人在地图上的唯一物理坐标”。无返回值（`void`），因为状态通过全局数组 `grid` 维护。
    
- **第 1 步与第 3 步 (终点与截断)：** DFS 会像树枝一样疯狂发散。一旦某条树枝找到了解（`r == MAX_ROW` 且 `solved = true`），后续所有的 `return` 会引发连锁反应，迅速终止其他无用树枝的生长。
    
- **第 4 步的边界陷阱：** 如果当前格子原本就有数字，必须向前走一步并立刻 `return`。如果忘记写 `return`，代码在跳过之后，还会强行往下执行 `for` 循环，覆盖掉原本合法的数字。
    
- **第 5 步的核心灵魂：** 回溯的本质是“打扫战场”。当你调用完 `dfs(r, c + 1)` 后，如果程序执行到了下面的 `grid[r][c] = 0`，这在逻辑上只说明一件事——**你刚才选的数字 `v` 在未来走进了死胡同。** 你必须把它抠出来，清空标记，让 `for` 循环转到 `v+1` 重新尝试。

## 模块五：今日高频盲区与灵魂自测

### 高频踩坑盲区

1. **数据结构的盲目信任：** 默认队列/栈里有东西，不写 `!empty()` 直接访问头部，导致考场喜提 RE（运行时错误）。
    
2. **多层循环的变量混用：** 在 `main` 函数里使用 `dfs` 里的局部变量，或者外层循环的 `i` 和内层循环的 `j` 在计算公式里写反。
    
3. **贪心策略以偏概全：** 没有用数据结构（哈希桶）通盘统计，而是随便抓一个当前元素当作“全局标准答案”。
    

### 灵魂自测 (建议遮挡下半部分回答)

**问题 1：**

在执行 `if(s[i] == q.front()) q.pop();` 时，如果遇到测试用例：`s = "A"`, `t = "A"`。当 `i = 1` 时（假设 `s` 后面还有空格被读取进来了），代码哪一行会爆出严重错误？应该如何用最简短的代码防范？

  
  
  
  
  

**答案 1：**

`q.front()` 会报错。因为当 `i=0` 时，'A' 已经匹配并 `pop` 掉了，此时队列为空。对空队列调用 `front()` 会触发未定义行为越界。防范方案：改为 `if(!q.empty() && s[i] == q.front())`。

**问题 2：**

在二维网格转一维存储时，假设我们要把一个长度为 15 的一维字符串，折叠成每行 5 个字符，共 3 行。请默写出：我要访问**第 2 行的第 4 列**（索引均从 0 开始），对应的一维字符串 `str` 的下标公式是什么？算出来的值是多少？

  
  
  
  
  

**答案 2：**

公式：`当前列 + 当前行 * 每行长度`。

带入数值：`4 + 2 * 5 = 14`。即 `str[14]`。

**问题 3：**

在使用哈希桶统计字符频次时，代码 `int count[26] = {0};` 应该写在外层循环（按列遍历）的**外面**，还是**里面**？如果写错了位置，会导致什么致命后果？

  
  
  
  
  

**答案 3：**

必须写在**外层循环的里面**。因为你要统计的是“当前这一列”的频次。如果写在外面，桶里的数据会不断累加，第 2 列的统计结果会混杂第 1 列的幽灵数据，导致局部贪心策略彻底崩溃。
