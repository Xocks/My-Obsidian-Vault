## 模块一：知识图谱与特定触发机制（Trigger）

- **算法 - 暴力枚举与降维映射**
    
    - **特定触发机制**：看到网格极小（如 6x6）、要求“不重叠选择”且选出特定形状（如 1x2 骨牌）。
        
    - **第一反应**：绝对不要用二维坐标做状态比较！立刻将二维坐标 `(r, c)` 映射为一维编号 `r * 总列数 + c`，将图形转换为集合，使用多重循环枚举并查重。
        
- **算法 - 数学与位运算**
    
    - **特定触发机制**：看到“逢二进一”、“合并翻倍”、“最终剩下1个”的字眼。
        
    - **第一反应**：这本质上是二进制拆分。判断一个数是否为 2 的整数次幂，无脑写出 $O(1)$ 模版 `(num & (num - 1)) == 0`。
        
- **数据结构 - 双端队列 (`deque`) 与取模优化**
    
    - **特定触发机制**：看到“胜者留队首，败者去队尾”，且**模拟的回合数 $K$ 极其巨大（如 10^9），远超数据量 $N$**。
        
    - **第一反应**：必有循环节！用 `deque` 模拟到最大值登顶后，直接用剩余回合数对 $N-1$ 取模，实现 $O(N)$ 降维打击。
        
- **算法 - DFS 与回溯**
    
    - **特定触发机制**：看到“网格填数”、“数独”、“不重复约束”、“输出任意一种解”。
        
    - **第一反应**：标准回溯模版。准备好布尔类型的 `visited` 数组（行、列、宫），严格执行“检查 $\to$ 标记 $\to$ 递归 $\to$ 撤销标记”四步走。
        
- **图论 - 置换环 (Permutation Cycle)**
    
    - **特定触发机制**：题目要求用“任意交换两项”的操作，将一个打乱的排列恢复为 `1 到 n` 的升序排列，求最少交换次数。
        
    - **第一反应**：建环！答案必定是 $N - \text{环的数量}$。

## 模块二：核心代码模版与保姆级拆解

以今天最具代表性的约束满足问题**“数字游戏 II (DFS回溯)”**为例：

**完整代码（C++）：**

C++

```
#include <bits/stdc++.h>
using namespace std;

int mtr[4][4];
bool ur[4][5];      // 记录某行是否用过数字 1-4
bool uc[4][5];      // 记录某列是否用过数字 1-4
bool ub[4][4][5];   // 记录某 2x2 宫是否用过数字 1-4

// 参数透视：r 代表当前准备填写的行号，c 代表当前准备填写的列号
void dfs(int r, int c) {
    // 1. 换行控制（越界处理）
    if(c == 4) {
        c = 0;
        r++;
    }
    
    // 2. 终止条件（胜利判定）
    if(r == 4) {
        for(int i = 0; i < 4; ++i) {
            for(int j = 0; j < 4; ++j) {
                cout << mtr[i][j];
                if(j != 3) cout << " ";
            }
            cout << '\n';
        }
        exit(0); // 找到第一组解直接拔电源，防止输出多组答案
    }

    // 3. 处理已知格子
    if(mtr[r][c] != 0) {
        dfs(r, c + 1);
        return; // 【关键】搜完后续必须返回，禁止执行下方填空逻辑
    }
    else {
        // 4. 处理空白格子：试探 1-4
        for(int k = 1; k <= 4; ++k) {
            // 剪枝：如果行、列、宫中有任何一个冲突，直接跳过
            if(ur[r][k] || uc[c][k] || ub[r/2][c/2][k]) {
                continue;
            }
            
            // 动作 A：做选择（打标记）
            mtr[r][c] = k;
            ur[r][k] = 1;
            uc[c][k] = 1;
            ub[r/2][c/2][k] = 1;
            
            // 动作 B：进入下一层
            dfs(r, c + 1);
            
            // 动作 C：撤销选择（回溯）
            mtr[r][c] = 0;
            ur[r][k] = 0;
            uc[c][k] = 0;
            ub[r/2][c/2][k] = 0;
        }
    }
}

int main() {
    ios::sync_with_stdio(false); cin.tie(0);
    for(int i = 0; i < 4; ++i) {
        for(int j = 0; j < 4; ++j) {
            cin >> mtr[i][j];
            if(mtr[i][j] != 0) {
                // 初始化录入时直接打好标记
                ur[i][mtr[i][j]] = 1;
                uc[j][mtr[i][j]] = 1;
                ub[i/2][j/2][mtr[i][j]] = 1;
            }
        }
    }
    dfs(0, 0);
    return 0;
}
```

**逐行剖析与参数透视**：

- `void dfs(int r, int c)`：没有返回值，依靠修改全局数组 `mtr` 运作。
    
- 边界条件控制：矩阵大小为 4x4，合法下标为 0 到 3。当 `c == 4` 时，说明当前行填满了，触发换行。当 `r == 4` 时，说明整个矩阵填满了，触发输出。
    
- `exit(0)` 的妙用：在多解题目中替代 `return`，确保只输出一种结果并瞬间终止进程，避免冗余回溯。

## 模块三：傻瓜式解题框架（模拟人类思考）

遇到任何网格填数（数独类）DFS 题目，闭上眼睛按照这三步默写：

- **第一步：怎么初始化？**
    
    建好存图的二维数组。更重要的是建好 **“检查锁”**（布尔数组）。读入数据时，只要发现不是 0，立刻把对应的行、列、块的布尔数组标为 `true`。
    
- **第二步：核心逻辑怎么写？**
    
    采用“蛇形探路”。
    
    - 当前是数字？跳过它，`dfs(下一格)`，然后**必须紧跟 `return`**。
        
    - 当前是空位？开启 `for` 循环遍历候选数字。查三把锁，全开才能进。填数、上锁、`dfs(下一格)`、下锁、清空。
        
- **第三步：怎么处理边界？**
    
    先处理越界换行（列指标越界），再处理终点胜利（行指标越界）。胜利时打印结果，如果是单解题写 `exit(0)` 结束战斗。

## 模块四：我的源码级纠错与降维打击

1. 源码级纠错：卡牌合并的逻辑运算符

**你的原代码逻辑：**

C++

```
else if(num == 2 && num == 1) // 你的错误写法
```

**一针见血**：`&&` 代表“并且”。一个变量不可能同时等于 1 又等于 2，这个分支永远无法触发。

**正确写法**：必须使用 `||`（或者），即 `num == 2 || num == 1`。

### 2. 源码级纠错：DFS 填数逻辑反转

**你的原代码逻辑：**

C++

```
int v = mtr[r][c];
if(v == 0) { dfs(r, c+1); return; } // 空白格子直接跳过
else { 
    for(int k=1; k<=4; ++k) {
        if(ur[r][v] != 0 ...) // 检查了已有数字 v，而不是准备填入的 k
```

**一针见血**：逻辑完全相反。`v == 0` 代表这里是空位，正是你需要通过 `for` 循环去填数的地方，你却跳过了它；对于已有数字，你反而去尝试修改它，并且检查标记时错误地使用了旧变量 `v` 而不是待填入的 `k`。

3. 降维打击：擂台轮转的 $O(K)$ 变 $O(N)$

**原思路痛点**：你使用了单纯的 `queue` 模拟回合数。当 $K = 10^9$ 时，`while(k--)` 需要执行十亿次，必定引发 Time Limit Exceeded (TLE)。

**降维替代算法**：**找极值 + 取模跃迁**。

- **大白话解释**：队伍里战斗力最高的人一旦当上擂主，就不可能被打败。剩下的回合，无非是其他所有人排队上来挨揍，然后滚去队尾。这是一个完全可预测的**死循环**。
    
- **时空优势**：用 `deque` 模拟到最高战力者登顶（最多只需 $N$ 步），剩下的 $K - steps$ 次对决，直接对剩余人数 $N-1$ **取模**。无论 $K$ 有多大，时间复杂度瞬间从 $O(K)$ 暴降至 $O(N)$，执行时间从可能超时压缩到 1 毫秒内。

## 模块五：抗遗忘复习卡片与灵魂自测

### 盲区总结

1. **网格遍历边界越界**：在 `for` 循环中访问 `j+1` 或 `i+1` 时，循环上限没有相应减一（如 6x6 矩阵，横向访问 `j+1` 时，`j` 必须 `< 5`）。
    
2. **DFS 缺少阻断**：在 DFS 遇到已知条件直接跳过并进入下一层后，忘记写 `return`，导致回溯阶段意外执行了后续覆盖代码。

## _(防剧透分割线)_

**自测答案：**

- **A1**: `return` 只会退出当前深度的函数调用，程序依然会回溯并继续寻找其它分支的解，导致输出多个答案。`exit(0)` 是直接终止整个进程，确保在打印出第一组正确答案后立刻停止。
    
- **A2**: 应该选择 `deque`（双端队列）。`vector` 的左端（队头）是封死的，不支持 `push_front` 操作。若在 `vector` 的最前面强行插入元素，会导致后续所有元素在内存中整体向右搬移，时间复杂度高达 $O(N)$。
    
- **A3**: `(num & (num - 1)) == 0`。
