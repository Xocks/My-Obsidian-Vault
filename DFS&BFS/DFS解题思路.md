### 第一步：扔掉代码，先画一棵“决策树”

当你拿到题目时，**千万不要一上来就去想怎么写 `dfs()` 函数，也不要想参数和结束条件。** 大脑的第一反应应该是：**如果我是个傻子，我要怎么通过一步步“做选择”把所有结果列出来？**

1. **起点在哪？**（初始状态）
    
2. **此时此刻，我面前有几条路可以选？**（选择列表）
    
3. **我选了其中一条路走下去后，我的处境发生了什么变化？**（状态转移）
    
4. **走到什么地步，我就无路可走，或者找到答案了？**（叶子节点/结束条件）
    

**举个例子（为什么参数不是层数而是数字）：**

在上面“自然数拆分 $n=7$”的题目中，大脑模拟的过程是这样的：

- **一开始：** 我手里有 $7$，我想把它拆开。
    
- **第一步做选择：** 我第一个数能选几？我可以选 1, 2, 3...
    
- **处境变化（核心！）：** 假设我第一步选了 $2$。现在我的处境变了！我的目标不再是拆分 $7$，而是变成了：**“用大于等于 $2$ 的数字，去凑齐剩下的 $5$ ($7-2$)”**。
    
- **继续做选择：** 面对凑齐 $5$ 的目标，我下一步能选几？我能选 2, 3。
    

你看，在这个思考过程中，“我在第几层（我选了几个数）”根本就不重要！重要的是**“我还剩多少没凑齐（`target_sum`）”**以及**“我这把最小能选几（`start_num`）”**。

### 第二步：如何入手题目？（思考顺序）

解答你的核心疑问：**先思考参数，还是先思考结束条件？**

**答案是：先思考“状态参数”，再思考“结束条件”。** 因为结束条件本质上就是“某一个特定的最终状态”。如果你连状态（参数）都没定义清楚，你根本不知道拿什么去判断结束。

#### 思考步骤框架（请死记硬背这个顺序）：

1. **定义 DFS 函数的参数（确定状态）：**
    
    - **问自己：** “我在树上的任意一个节点时，我需要知道哪些信息，才能决定下一步怎么走？”
        
    - 如果是排列组合（位置相关），你需要知道当前填到了第几个格子，所以参数是 `level` 或 `index`。
        
    - 如果是拆分数字，你需要知道还剩多少没拆完，以及现在能选的最小数字，所以参数是 `target_sum` 和 `start_num`。
        
    - 如果是走迷宫，你需要知道你当前站在哪个坐标，所以参数是 `x` 和 `y`。
        
    - **写出第一步：** 确定函数签名，例如 `void dfs(int target_sum, int start_num)`。
        
2. **设计结束条件（Base Case）：**
    
    - **问自己：** “看着参数，当这些参数变成什么样的时候，我就该停下来了？”
        
    - 剩余数字被扣减到了 0？(`if target_sum == 0`)
        
    - 走到了第 $n$ 个格子？(`if level == n`)
        
    - 走到了迷宫的出口坐标？(`if x == end_x && y == end_y`)
        
    - **写出第二步：** 在 `dfs` 函数的第一行写下 `if (...) { ... return; }`。
        
3. **写出单层搜索逻辑（For 循环与递归）：**
    
    - **问自己：** “在当前这个状态下，我能做哪些合法的选择？”
        
    - 用 `for` 循环遍历所有合法选择。
        
    - 做出选择 -> 递归进入下一个状态（参数发生改变！） -> 撤销选择（回溯）。

### 第三步：DFS 到底能不能归类？

**绝对可以归类。** 很多时候你觉得需要“随机应变”，是因为你还没见过足够多的题型。在算法竞赛中，常见的 DFS 基本跑不出这四大类，每一类都有自己固定的**参数特征**：

|**DFS 题型分类**|**核心目标**|**典型的 dfs() 参数长什么样**|
|---|---|---|
|**1. 组合/排列/子集**|挑选元素、全排列|`dfs(int index)`<br><br>  <br><br>知道当前在处理数组的第几个位置。|
|**2. 约束满足问题**<br><br>  <br><br>(如：N皇后，数字拆分)|满足特定条件下的分配|`dfs(int remaining, int current_limit, ...)`<br><br>  <br><br>知道还剩多少任务，当前的限制是什么。|
|**3. 网格/矩阵搜索**<br><br>  <br><br>(如：走迷宫，岛屿数量)|在二维地图上蔓延|`dfs(int x, int y)`<br><br>  <br><br>知道当前所处的二维物理坐标。|
|**4. 树/图的遍历**|探索节点连通性|`dfs(Node* curr, int parent)`<br><br>  <br><br>知道当前访问到了哪个节点。|

只要你能识别出这道题属于哪一类，你的大脑就会立刻弹射出对应的参数模板。你觉得“传入数字而不是层数”很奇怪，只是因为你刚刚从第 1 类（组合）跨入到了第 2 类（约束满足问题），其实它们在本质上都是在**探索状态空间**。

### 总结你的大脑运作流程

下次拿到一道全新的搜索题，请强制自己按这个流程在大脑中过一遍：

1. **画树**：先在草稿纸上画出根节点和第一层的分支，搞清楚“选择”是什么。
    
2. **定状态（参数）**：从父节点走到子节点，什么变量改变了？把这些变量写进 `dfs(...)` 的括号里。
    
3. **写终止**：看着括号里的变量，想一下它们变成什么样就代表到底了。
    
4. **套模板**：把 `for` 循环、递归调用和回溯填进去。
