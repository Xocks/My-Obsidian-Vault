### 模块一：本周知识全景大纲（分类总结）

本周你攻克了 4 大核心算法簇，共计 10 道典型题目。

- **1. 深度优先搜索与回溯 (DFS) 类**
    
    - **指数型枚举（选或不选）：** B3622 枚举子集。核心特征：每个物品只有两个独立分支，不用 `for` 循环。适用场景：求所有可能组合的子集。
        
    - **排列型枚举（位置找人）：** B3623 枚举排列。核心特征：用 `for` 循环遍历候选人，用 `used` 数组标记。适用场景：求全排列、决定不同坑位的归属。
        
    - **组合型枚举与剪枝：** P10448 组合型枚举、烤鸡。核心特征：引入起始点变量强制递增防重复，利用剩余元素数量进行极值剪枝。适用场景：需要中途打断、限制长度或总和的组合穷举。
        
    - **全排列内置函数：** 免费披萨。核心特征：使用 `std::next_permutation`。适用场景：不需要中途剪枝、纯粹求所有顺序组合的多重集问题。
        
- **2. 数学逻辑与贪心类**
    
    - **极限条件推导：** 修剪灌木。核心特征：将过程模拟转化为物理距离的绝对值公式。适用场景：求周期性往返移动的最值问题。
        
    - **网格坐标映射：** 移动距离。核心特征：一维序号转二维 `(x, y)` 坐标，0 基准取模。适用场景：矩阵蛇形排号、曼哈顿距离计算。
        
    - **不变量与贪心平分：** 数组平分操作。核心特征：寻找操作前后的“总和不变量”，目标值锁定为向上取整的平均数。适用场景：每次操作一加一减求最小步数。
        
- **3. 字符串处理与底层输入输出**
    
    - **混合输入流清洗：** 航班时间。核心特征：处理带有回车的混杂输入。适用场景：`cin` 与 `getline` 连用时的缓冲区排雷。
        
    - **格式化提取：** 航班时间。核心特征：利用 `sscanf` 和 ASCII 偏移量解码。适用场景：处理时分秒格式的复杂字符串化简为绝对秒数。
        
- **4. 基础模拟、状态机与图论脑筋急转弯 (Ad-hoc)**
    
    - **时间步进与 LCM 优化：** P12170 攻击次数。核心特征：`while` 循环模拟回合。适用场景：直观过程推演，极大时转最小公倍数除法。
        
    - **滚动变量状态机：** P10899 劲舞团。核心特征：用两个变量记录当前与上次状态。适用场景：只需要比对相邻历史、优化 `O(N)` 空间到 `O(1)`。
        
    - **连通性分类讨论：** 不相邻元素交换。核心特征：画图验证 n=1,2,3,4 的死角。适用场景：看似数组交换，实则是图论节点是否全连通的推导。

### 模块二：极简解题框架（傻瓜式操作指南）

**题型一：DFS 回溯搜索框架**

1. **第一步（初始化与传参）：** 确定递归函数传什么参数（通常是：当前步骤/层数 `step`、当前累积状态 `sum`、目标值 `target`）。
    
2. **第二步（写死胡同与终点）：** 先写剪枝 `if(发现不可能) return;`，再写终点 `if(step == 终点) { 记录答案; return; }`。
    
3. **第三步（做选择与回溯）：** 写一个 `for` 循环遍历当前能选的选项 -> 记录当前选择（标记占用） -> 往下递归 `dfs(step + 1, ...)` -> 撤销选择（恢复标记）。
    

**题型二：二维网格坐标映射框架**

1. **第一步（降维归零）：** 拿到输入的所有编号，无脑先减 1（`m--`, `n--`）。
    
2. **第二步（算行列）：** 行号 `x = m / 宽度`，列号 `y = m % 宽度`。
    
3. **第三步（处理蛇形翻转）：** `if(x 是奇数)`，列号反转 `y = 宽度 - 1 - y`。最后直接求 ∣x1−x2∣+∣y1−y2∣。
    

**题型三：全排列组合枚举框架**

1. **第一步（必须排序）：** 拿到数组先 `sort(arr.begin(), arr.end());`。
    
2. **第二步（Do-While 循环）：** 写 `do { ... } while(next_permutation(arr.begin(), arr.end()));`。
    
3. **第三步（内部验算）：** 在 `do` 的括号里，把当前排列好的数组拼接成你需要的值，去打擂台求最大/最小值。

### 模块三：全量知识点与核心思路精讲

- **深搜中的“选与不选” vs “位置找人”：** 底层逻辑是你在对什么提问。如果你盯着“物品”，问的是“你要不要进背包？”，那就是指数型枚举，没有 `for`，只有两条平行的路。如果你盯着“坑位”，问的是“谁来坐这里？”，那就是排列型枚举，必须用 `for` 把所有人问一遍。
    
- **状态机的滚动变量空间优化：** 底层目的是省内存。当题目要求你判断“当前连击是否在 1000ms 内”，你没必要存下十万次敲击的所有时间。你只需要记住“上一秒”是谁，现在是谁。过去的数据没有利用价值，直接覆盖。
    
- **向零取整与向上取整的数学本质：** 编程语言底层用二进制除法，负数除以正数默认是砍掉小数部分靠近 0。这在求“平均目标值”时是致命的，因为数学上的平分需要向大的方向凑，这就是为什么必须区分正负号进行绝对精确的数学重构。
    
- **图论连通性的 Ad-hoc 思维：** 这类脑筋急转弯的核心思路是找“瓶颈”。别去想怎么交换，去想“谁动不了”。当元素距离必须大于 1 才能换，中间的元素就成了孤岛。一旦元素增加，外围元素就成了跳板，孤岛也就连通了。

### 模块四：源码级纠错与降维打击

这是你本周踩过的坑，我完整保留了你的变量名，指出红线区：

**错误 1：负数除法陷阱（数组平分操作）**

C++

```
// 你的原版错误逻辑：
ll all = ...; // 可能算出来是负数
ll avg = (all + n - 1) / n; // ❌ 错误点：当 all 是 -2，n 是 3 时，算出来是 0，公式失效。

// 源码级修正方案：
ll avg;
if (all >= 0) {
    avg = (all + n - 1) / n;
} else {
    avg = all / n; // ✅ 纠错点：直接用 C++ 自带的负数向 0 取整机制，对于负数这就是向上取整。
}
```

**错误 2：`if` 作用域裸奔（不相邻元素交换）**

C++

```
// 你的原版错误逻辑：
for(int i = 1; i < n; i++) {
    if(arr[i] < arr[i-1]) is = 0; break; 
    // ❌ 错误点：break 脱离了 if 的控制！这导致循环只执行 1 次就绝对会撞上 break 退出。
}

// 源码级修正方案：
for(int i = 1; i < n; i++) {
    if(arr[i] < arr[i-1]) {
        is = 0; 
        break; // ✅ 纠错点：必须加 {} 把动作包起来。
    } 
}
```

**错误 3：ASCII 运算未减基准（航班时间）**

C++

```
// 你的原版错误逻辑：

// ❌ 错误点：如果输入是 '(+1)'，tim[20] 是字符 '1'，ASCII 码是 49。你实际加了 24 * 49。
```

**降维打击对比区：**

1. **《修剪灌木》**
    
    - **你的算法：** 开一个 `O(N)` 数组预处理一半，判断奇偶性倒序输出。
        
    - **替代算法（降维）：** 直接用一个 `for` 循环，里面写 `cout << max(i, n - 1 - i) * 2`。
        
    - **为什么更好：** 空间复杂度从 `O(N)` 降为 `O(1)`，代码少写 15 行，彻底屏蔽奇偶数中心点判断失误的 bug。

    - **替代错觉：** 逆向推导找 n 的因数。最坏时间复杂度 O(n)，对于千万级数据稳稳超时。
        
    - **正解算法（降维）：** 也就是你后来做对的，直接用 `next_permutation`。时间复杂度被严密封锁在排列种数级别（常数时间操作），空间 `O(1)`。这就是不用绕路、直接枚举的降维碾压。

**模版 2：带严格剪枝的 DFS 组合型枚举框架**

C++

```
#include <iostream>
#include <vector>
using namespace std;

vector<int> path; // 存放路径
int n, k; // 从 n 个数选 k 个

// 函数传参：
// st: 已经选了多少个数（坑位级数）
// num: 当前正准备考虑选哪个数字（候选人编号）
void dfs_pick(int st, int num) {
    // 【避坑指南】：终点判断的 return 必须在最前面，且包裹在 if 内部
    if (st == k) {
        // 输出 path 或者进行验证
        return; 
    }
    
    // 越界保护
    if (num > n) return;
    
    // 【终极剪枝核心】：
    // n - num + 1 = 剩下的候选人数量
    // k - st = 还需要填的坑位数量
    // 剩下的不够填了，直接结束递归，省下巨量时间！
    if (n - num + 1 < k - st) return;

    // 决定 1：选当前数字 num
    path.push_back(num);        // 动作 1: 占坑
    dfs_pick(st + 1, num + 1);  // 动作 2: 往下一层走，坑位+1，候选人+1
    path.pop_back();            // 动作 3: 撤销占坑，回溯现场

    // 决定 2：不选当前数字 num
    dfs_pick(st, num + 1);      // 坑位不变，候选人换成下一个继续试
}

int main() {
    n = 5, k = 3;
    dfs_pick(0, 1); // 0 个坑位已填，从数字 1 开始考虑
    return 0;
}
```

### 模块六：复习自测‘灵魂拷问’（高效复习机制）

请遮住下方答案进行自我测试：

**自测问答题区**

1. **灵魂拷问一：** 在处理二维网格坐标映射或者周期取模问题时，拿到 1 为起点的编号，你的第一手本能操作应该是什么代码？
    
2. **灵魂拷问二：** 当我写出 `cin >> a; string s; getline(cin, s);` 时，程序绝对会发生什么致命错误？怎么修？
    
3. **灵魂拷问三：** 我用 DFS 穷举一个总和上限为 `target` 的问题，如果想做“可行性剪枝”，这行 `if( ??? ) return;` 代码应该填什么？
    
4. **灵魂拷问四：** 在 C++ 算法里算两个变量的平均数（要求严格向上取整），如果这两个变量的总和可能是负数，为什么绝对不能写 `(a+b-1)/b`？
    
5. **灵魂拷问五：** 在调用全排列神器 `std::next_permutation(arr.begin(), arr.end())` 之前，如果不加什么前置操作，会直接漏掉一大半的答案？
    
6. **灵魂拷问六：** 为什么在写 `if` 语句时，只要后续跟着超过 1 条执行命令，哪怕是给变量赋值再 `break`，也死活必须加上大括号 `{}`？
    

**自测答案区**

1. **答：** 立刻 `m--; n--;`，全面降维到 0 基准，算完所有行列和取模后再加回去。
    
2. **答：** `getline` 会直接截获 `cin` 遗留下来的回车符 `\n`，读入一个空字符串导致后续越界崩盘。修法：中间插入 `cin.ignore(10000, '\n');`。
    
3. **答：** 填 `当前已经累加的和 > target` 或 `当前和 + 剩下全选最小的 > target`。这能提前砍掉绝对超标的废树枝。
    
4. **答：** 因为 C++ 底层的负数除法是向 0 取整。例如 -2 / 3 会变成 0，导致公式逻辑完全失效。必须写 `if(总和 >= 0)` 分支来处理。
    
5. **答：** 必须先 `sort(arr.begin(), arr.end());` 让它从最小字典序开始，配合 `do...while` 循环使用。
    
6. **答：** 不加大括号，`if` 只会管辖紧贴着它的第一条语句。后面的 `break` 会脱离控制，变成每次循环必执行的无条件强制中断，导致只跑一次就全盘死亡。
