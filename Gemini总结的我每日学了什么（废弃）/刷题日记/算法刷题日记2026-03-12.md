### 一、 核心考点与思路归纳

今天涉及的四大核心考点及通用的标准思考步骤如下：

1. **找规律与极值推导（修剪灌木）**
    
    - **考点：** 将“过程模拟”转化为“物理距离计算”。
        
    - **思考步骤：** 遇到周期性或往返移动的问题，不要按时间轴一步步模拟。直接寻找影响结果的**极端条件**（例如：距离最远的端点），利用公式 $\max(左距离, 右距离) \times 2$ 一步到位求出结果。
        
2. **二维矩阵与曼哈顿距离（移动距离）**
    
    - **考点：** 一维序号向二维坐标 $(x, y)$ 的映射，以及蛇形折叠处理。
        
    - **思考步骤：** 凡是遇到“矩阵排号”、“网格移动”求距离，**第一步永远是将序号整体减 $1$（转为 0 基准）**。第二步通过 `/` 求行号，`%` 求列号。第三步判断奇偶行进行反向翻转。第四步直接求坐标差绝对值之和。
        
3. **字符串解析与时间映射（航班时间）**
    
    - **考点：** 混合输入流的缓冲区处理，以及跨天时间的统一计量。
        
    - **思考步骤：** 遇到时、分、秒计算，**绝对不要尝试借位换算**。将所有时间统一换算为“距离零点的总秒数”。遇到包含特定格式的复杂字符串，优先考虑格式化读取工具，而非手工切片。
        
4. **深度优先搜索与回溯（烤鸡）**
    
    - **考点：** 全排列/组合的穷举，状态树的遍历与剪枝。
        
    - **思考步骤：** 画出决策树。明确三个核心要素：**当前所处阶段（Step）**、**当前累积状态（Sum/Cur）**、**终止与剪枝条件**。严格遵循“做选择 -> 递归 -> 撤销选择”的闭环。

### 二、 错误诊断与避坑指南

结合你今天手敲的代码逻辑，以下是你最容易踩坑的 4 个“雷区”，请在以后的编码中特别留意：

- **雷区 1：1-based 索引的取模陷阱**
    
    - **误区：** 在《移动距离》中，直接对原序号 `m % w`。这会导致每一行的最后一个元素余数为 0，发生“跳行”或“归零”的逻辑错乱。
        
    - **避坑：** 处理一切周期、矩阵、取模问题，**先无脑 `m--; n--;`** 降维到 0 基准，算完如果需要再加回去。
        
- **雷区 2：字符的 ASCII 码运算陷阱**
    
    - **误区：** 在《航班时间》中写出 `h2 += 24 * tim[20]`。此时如果输入是 `(+1)`，提取到的字符是 `'1'`，其底层 ASCII 码为 49，实际运算变成了 `+ 24 * 49`。
        
    - **避坑：** 字符转数字必须减去零的字符基准值，即 **`tim[20] - '0'`**。
        
- **雷区 3：混合读取的缓冲区污染**
    
    - **误区：** 使用 `cin >> m;` 后直接调用 `getline(cin, str);`。`cin` 留下的换行符 `\n` 会被 `getline` 直接截获，导致读入空字符串，程序崩溃越界。
        
    - **避坑：** 只要前面用了 `cin >>` 且后面要用 `getline`，中间必须插入清道夫：`cin.ignore(10000, '\n');`。
        
- **雷区 4：DFS 的 `return` 位置与状态叠加**
    
    - **误区：** 在《烤鸡》中，将 `return;` 写在了终止判断 `if` 的外部，导致递归直接死亡；并在传参时写成了 `cur + cur` 而不是 `cur + i`。
        
    - **避坑：** 递归的 `return` 必须严格包裹在边界判断的 `if` 内部。递归向下传递的参数应当是“旧状态 + 当前新选择的值”。

### 三、 更优解法与复杂度对比

- **《修剪灌木》：空间 $O(N)$ 优化至 $O(1)$**
    
    - **你的原始思路：** 开辟数组预先计算一半的值，通过奇偶判断倒序输出。
        
    - **更优解法：** 直接在循环内计算 $\max(i, N - 1 - i) \times 2$ 并输出。
        
    - **优势：** 抛弃了数组，节省了内存；彻底消灭了偶数中心点重复的边界 bug。
        
- **《航班时间》：手工切片优化为 `sscanf` 模版**
    
    - **你的原始思路：** `(tim[0]-'0')*10 + tim[1]-'0'`... 繁琐地计算每个位置的索引并相加。
        
    - **更优解法：** 使用 C++ 标准库的 `sscanf` 按照模版串一次性提取。
        
    - **优势：** 代码从 20 行缩减至 2 行，且无需手动判断字符串长度，极大降低了数错索引导致越界的风险。

**模版 2：复杂格式化字符串提取 (`sscanf`)**

C++

```
string line;
getline(cin, line);

int h1, m1, s1, h2, m2, s2, days = 0; // days 必须赋初值 0
// sscanf 可以直接根据格式从字符串中抠出对应的整数
// 如果没有 (+d)，它就只提取前 6 个，days 依然安全地保持为 0
sscanf(line.c_str(), "%d:%d:%d %d:%d:%d (+%d)", &h1, &m1, &s1, &h2, &m2, &s2, &days);
```

**模版 3：DFS 回溯搜索通用框架**

C++

```
vector<vector<int>> res; // 存放所有合法结果
vector<int> path;        // 存放当前的单条路径

// st: 当前所在的层级/步骤; cur: 当前的累积状态; target: 目标值
void dfs(int st, int cur, int target) {
    // 1. 极简剪枝 (提前排除绝对不可能的情况)
    if (cur > target) return; 
    
    // 2. 终止边界条件
    if (st > 10) { 
        if (cur == target) res.push_back(path); // 收集正确结果
        return; // 【注意】必须在边界判断内部 return
    }
    
    // 3. 遍历当前层级的所有选择
    for (int i = 1; i <= 3; ++i) {
        path.push_back(i);               // 做出选择
        dfs(st + 1, cur + i, target);    // 递归进入下一层
        path.pop_back();                 // 回溯：撤销选择，恢复现场
    }
}
```
