### 模块二：无代码解题骨架（只讲人类思考过程）

**核心题型：网格填数约束问题（DFS回溯）**

- **第一步：怎么初始化？** 除了建立存放数字的棋盘二维数组，**必须**同步建立多把“布尔检查锁”（如记录某行、某列、某宫是否已经用过数字 X 的数组）。在读入初始残局数据时，只要发现数字不是 0，立刻把对应的锁扣上（设为 true）。
    
- **第二步：核心逻辑是什么？** 指针按“蛇形”遍历每个格子。
    
    遇到已知数字：直接跨过它，递归进入下一格，**紧接着必须强行阻断当前函数**。
    
    遇到空位：开启循环，拿着候选数字去试探。试探前查锁，只有所有锁都没锁上，才能执行四步曲：填入数字并上锁 $\to$ 带着这个状态递归下一格 $\to$ 退回来时解开所有锁 $\to$ 把格子重新清空。
    
- **第三步：输出前需要注意什么特殊的边界？** 优先判断“列越界”：当列指针走出最右侧，强行将其拉回第 0 列，同时行指针加一。
    
    其次判断“行越界”：当行指针走出最底侧，说明网格全填满了（触发胜利条件）。由于题目只要求一种解，打印完盘面后，千万别让它原路返回，直接拔电源结束整个程序。

### 模块三：点睛之笔与我的高价值错题本

#### 1. 提纯核心代码（神级模版）

- **$O(1)$ 判定 2 的次幂**：`if ((num & (num - 1)) == 0)`
    
    _原理_：2 的次幂二进制里只有一个 `1`。减一后，唯一的 `1` 变 `0`，后面的 `0` 全变 `1`。两者按位与，必然完美错开得 0。
    
- **二维转一维绝杀公式**：`id = r * 总列数 + c`
    
    _原理_：在做图形查重时，彻底抛弃繁琐的 `x, y` 比较，直接化为单一整数的集合冲突判定。
    
- **DFS 状态撤销核心**：`ur[r][k] = 1; dfs(r, c+1); ur[r][k] = 0;`
    
    _原理_：做选择，去未来的世界看看；发现走不通退回来时，必须把当初做的标记擦得一干二净，假装无事发生，去试下一个可能。
    

#### 2. 源码级精准纠错

- **《卡牌合并》逻辑符硬伤**
    
    - 你的原代码：`else if(num == 2 && num == 1)`
        
    - **一针见血**：`&&` 是“并且”。一个变量不可能同时等于 1 和 2！这里永远无法触发，想表达“满足其一即可”，必须用 `||`（或者）。
        
- **《数字游戏 II》填数逻辑反转与控制流缺失**
    
    - 你的原代码：`if(v==0) { dfs(r,c+1); return; }` 然后在 `else` 里 `for(int k = 1;k<=4;++k) { if(ur[r][v]!=0...`
        
    - **一针见血**：全盘写反了！`v==0` 是空位，本该去循环填数你却跳过了；`else` 是有数字的格子，你却跑去填数，还用旧数字 `v` 去查重（应该查待填入的 `k`）。此外，当你修好逻辑，在处理 `v != 0` 的格子并调用 `dfs` 之后，**少写了 `return`**，导致退回来时覆写了已有数字。
        
- **《矩阵染色》二维偏移量越界**
    
    - 你的原代码：纵向收集时 `for(int i=0;i<6;++i) { for(int j=0;j<5;++j) { if((mat[i][j]+mat[i+1][j])%2!=0)`
        
    - **一针见血**：边界搞混了！逆推法：因为下面出现了 `i+1`，所以 `i` 必须 `< 5` 防止行越界；而列坐标只是单纯的 `j` 没有加一，所以 `j` 必须 `< 6`，你写成 `< 5` 直接导致漏搜了矩阵最右侧整整一列的纵向骨牌！
        

#### 3. 降维打击（时空质变）

- **《擂台轮转》**：暴力 `queue` 模拟时间复杂度 $O(K)$（面对十亿次直接 TLE） $\to$ 降维算法：**找极值 + 取模跃迁**，时间复杂度暴降至 $O(N)$。
    
    _优势_：由于最大值一旦排到队头就进入不败的死循环，直接拿剩余回合数对剩余人数取余，跳过中间毫无意义的挨揍过程，瞬间算出最终状态。
