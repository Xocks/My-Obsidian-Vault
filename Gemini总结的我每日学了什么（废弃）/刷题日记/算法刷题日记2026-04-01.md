### 模块三：傻瓜式解题框架（模拟人类思考）

**题型一：矩阵 DFS 回溯填数**

- **第一步：定义状态与字典**。确立当前走到哪一格（坐标 `r, c`），并准备三个“查重字典”（行、列、子块），用来记录哪个数字已经被用过了。
    
- **第二步：核心逻辑（分层安检与试错）**。
    
    - 安检阶段：列越界了，直接把任务踢给下一行的开头；行越界了，说明填满了。
        
    - 干活阶段：遇到填好的格子直接跳过。遇到空格，就翻开字典，把 1∼4 中没冲突的数字填进去，并在字典上打勾，然后走向下一格。
        
    - 后悔阶段（极度关键）：从下一格退回来时，**必须把刚才填的数字擦掉，字典上的勾也必须抹掉**，干干净净地去尝试下一个数字。
        
- **第三步：处理输出边界**。当“行安检”发现指针已经越过最后一行时，说明找出了一个完全合法的局面，立刻打印整个矩阵，并使用 `exit(0)` 强制结束整个程序，防止它继续寻找多余的解。
    

**题型二：极坐标几何贪心寻路**

- **第一步：确立目标参数**。利用勾股定理算出终点所在的绝对极径（最大半径 R），利用纵横坐标直接算出它与 X 轴的绝对弧度 θ。
    
- **第二步：核心贪心推导**。因为沿着 X 轴走能“免费”撑大我们的半径，而在越大的圆上转圈，能跨越的坐标跨度越大。所以最省力的走法是：开局直接沿 X 轴狂奔到最大半径 R 的位置，然后一口气顺着大圆弧转到终点。
    
- **第三步：处理输出**。总距离 = 水平走的直线距离（R） + 圆弧的长度（R×θ）。最后必须对这个浮点数结果进行四舍五入。

### 模块四：逻辑切片与源码纠错（核心精华）

**1. 灵魂 5 行（DFS 防越界与回溯的最强骨架）**

C++

```
if(c == 4) { dfs(r+1, 0); return; } // 列安检：溢出则强制甩锅给下一行，并截断本层
if(r == 4) { exit(0); }             // 行安检：溢出说明全部填满，直接结束
// ... 查字典判断无冲突后 ...
ur[r][k] = uc[c][k] = ub[r/2][c/2][k] = 1; // 标记占用（锁门）
dfs(r, c+1);                               // 深入探索
ur[r][k] = uc[c][k] = ub[r/2][c/2][k] = 0; // 回溯（开门）：撤销标记，为其他路线让路
```

_大白话解释_：前两行是函数的“门神”，各司其职，保证你永远不会带着非法的坐标往下去读数组。后三行是回溯的命脉，进去试探前留下脚印，试探无果退回来时，必须把脚印扫干净，否则别的分身过来会以为这条路不通。

**2. 精准手术级纠错**

**【错题一：DFS 越界炸弹】**

C++

```
if(c==4) {
    c = 0;
    r++;
}
int v = mtr[r][c]; 
// 🚨 我哪一步想错了：当遍历到最后一行的越界列 (r=3, c=4) 时，你在此处将 r 强行自增变成了 4。紧接着下一句代码拿着 r=4 去读 mtr[4][0]，数组最大只有 [3][3]，当场越界崩溃！
// 修正逻辑：当前层只管当前层。c==4 时，直接调用 dfs(r+1, 0); 然后立刻 return。绝不能在本层擅自修改 r 的值再让代码往下跑。
```

**【错题二：极坐标的数学陷阱】**

C++

```
double a = asin(double(666/233)); 
// 🚨 我哪一步想错了：
// 陷阱 1：666/233 是两个 int 相除，C++ 会直接无情截断得到整数 2。
// 陷阱 2：y/x 算出来的是正切值(tan)，你把它传给了反正弦(asin)，且 asin(2) 超过了取值范围 [-1, 1]，会直接报错返回 nan。
// 修正逻辑：直接改为 double a = atan2(666, 233);

cout << (int)b; 
// 🚨 我哪一步想错了：(int) 强制转换的本质是向下取整。无论小数是 .1 还是 .9，统统抹掉。这违背了题目“四舍五入”的要求。
// 修正逻辑：改为 cout << (long long)round(b);
```

**3. 降维打击** 在坐标系限定规则寻路题中，如果使用二维网格 BFS 一步步搜，时间复杂度是极其恐怖的 O(N2) 且小数状态极难维护；而采用**极坐标贪心（路径 = R+R×θ）**，时间复杂度直接降维至 O(1)，纯数学 O 秒出解。

### 模块五：抗遗忘灵魂自测（主动回想）

**今日最易踩坑盲区总结：**

1. **越权操作**：在递归中总想“越俎代庖”，喜欢在一层函数里修改本该属于下一层处理的坐标（如 `c=0; r++`），导致边界拦截器完全失效。
    
2. **隐式类型截断**：对 C++ 的整数除法极度不敏感，总是让表达式先算出被截断的整数结果后，才想起来套个 `(double)` 补救，但此时精度已死。
    

**🔥 直击灵魂的自测题：**

1. 在网格 DFS 中，如果在 `for` 循环里做完 `dfs(r, c+1)` 的深入探索后，忘记写 `ur[r][k] = 0;` （抹除标记）的代码，程序在遇到“死胡同”退回来尝试其他分支时，会发生什么致命现象？
    
2. 已知坐标为 (0,5)，如果错误地使用 `atan(5 / 0)` 会引发什么程序级别的灾难？为什么换成 `atan2(5, 0)` 就能完美运行？
    
3. 在代码 `double res = 5 / 2;` 中，最终 `res` 打印出来的值是多少？为了得到真实的小数，我应该把等号右边修改成什么样？
    

**【自测答案区】**

1. **答案**：程序会认为这个数字 k 依然“被占用着”。当它退回上一步尝试寻找其他合法解时，同级别的循环会发生误判，最终导致整个网格死锁，找不到任何存在的正确解。
    
2. **答案**：`atan(5 / 0)` 会立刻触发“除以零错误（Division by Zero）”，导致程序在运行时直接崩溃报 RE。而 `atan2(y, x)` 是底层优化过的二元函数，它能识别出 x=0 的情况，直接安全地返回 π/2（90度），并且还能准确识别象限。
    
3. **答案**：最终打印出来的值是 `2.0`。因为 `5` 和 `2` 都是整型，相除时小数部分直接被腰斩抛弃，得到 `2` 后才转换成 `double`。要得到真实小数，必须写成 `5.0 / 2.0`（或者 `(double)5 / 2`）。
