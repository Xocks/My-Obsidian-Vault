## 模块三：我的源码级纠错与降维打击

### 1. 《观星》（连通块）—— 基础方向与越界盲区

**原味纠错**：

C++

```
// 【错误1】题目说明相邻是 8 个位置，你只写了上下左右 4 个。
int xm[4] = {1,0,-1,0}; 
int ym[4] = {0,1,0,-1};

int bfs (int a,int b) // 实际上是 DFS 逻辑
{
    // 【错误2】致命的越界漏洞。如果 a 是 -1，去查 fl[-1][b] 会直接引发段错误导致程序崩溃。
    // 必须在访问数组前，先判断 a < 0 || a >= n || b < 0 || b >= m。
    if(fl[a][b]==1||arr[a][b]==0)return 0; 
    
    int res = 0; // 【错误3】当前这颗星星本身要算作 1，res 应该初始化为 1。
    for(int i = 0;i<4;++i)
    {
        int newx = a + xm[i];
        int newy = b + ym[i];
        // 【错误4】行列判断混淆。n 是行，m 是列。应该是 newx < n && newy < m。
        if(newx>=0&&newx<m&&newy>=0&&newy<n)
        {
            fl[newx][newy]++; // 【错误5】标记时机错误。进入函数的第一时间就该标记，而不是在这里。
            // 【错误6】因为 res 初始为 0，每次递归 +1 会导致子节点数量被错误放大。
            res = res + 1 + bfs(newx,newy); 
        }
    }
    return res;
}
```

### 降维打击（针对网格搜索）：BFS 广度优先搜索

**替代原因**：对于 $1500 \times 1500$ 且满屏都是星星的极端测试用例，DFS 递归深度会达到惊人的 $2.25 \times 10^6$ 层。C++ 默认系统栈只有区区 8MB 左右，必定会**栈溢出 (Runtime Error)**。

**降维解法**：使用 `std::queue` 实现 BFS。它将递归转移到堆内存中，堆内存有几个 GB 的空间，极其稳定，绝对不会爆栈。

## 模块四：核心代码模版与保姆级拆解

为你提取防爆栈的 **BFS 网格连通块搜索模版**，这是竞赛工程师的标准起手式。

### 完整代码与详细注释 (使用新变量名以示区别)

C++

```
#include <bits/stdc++.h>
using namespace std;

const int MAX_ROW = 1600;
const int MAX_COL = 1600;
int grid[MAX_ROW][MAX_COL];   // 0代表空地，1代表有效节点
bool visited[MAX_ROW][MAX_COL]; // false代表未访问，true代表已访问
int rows, cols;

// 8个方向的偏移量
int dx[8] = {-1, -1, -1, 0, 0, 1, 1, 1};
int dy[8] = {-1, 0, 1, -1, 1, -1, 0, 1};

// 模版函数：以 (start_r, start_c) 为起点探索整个连通块，返回节点总数
int bfs_get_component_size(int start_r, int start_c) {
    // 1. 初始化队列，并将起点推入
    queue<pair<int, int>> q;
    q.push({start_r, start_c});
    
    // 2. 极其重要：只要进入队列，必须立刻标记为已访问！防重复入队
    visited[start_r][start_c] = true;
    
    int component_size = 0; // 记录连通块大小
    
    // 3. 只要队列不空，说明波纹还在扩散
    while (!q.empty()) {
        pair<int, int> curr = q.front(); // 获取队首
        q.pop(); // 弹出队首
        
        component_size++; // 每弹出一个真实节点，连通块大小加1
        
        // 4. 尝试向周围 8 个方向扩散
        for (int i = 0; i < 8; ++i) {
            int nr = curr.first + dx[i];
            int nc = curr.second + dy[i];
            
            // 5. 边界防线：不越界 && 是有效节点 && 没被访问过
            if (nr >= 0 && nr < rows && nc >= 0 && nc < cols 
                && grid[nr][nc] == 1 && !visited[nr][nc]) {
                
                visited[nr][nc] = true; // 立刻标记！
                q.push({nr, nc});       // 压入队列排队等待后续扩散
            }
        }
    }
    return component_size; // 队列空了，返回探索完毕的总大小
}
```

### 逐行剖析与参数透视

- **`queue<pair<int, int>> q;`**
    
    - **作用**：建立一个排队系统。`pair` 的 `first` 存行坐标，`second` 存列坐标。
        
- **`visited[start_r][start_c] = true;`**
    
    - **作用**：烙下印记。**边界防线核心**：如果在 `push` 进队列时不立刻打上标记，其他节点扩散时可能会把这个点再次扔进队列，导致死循环卡死。
        
- **`nr >= 0 && nr < rows && nc >= 0 && nc < cols`**
    
    - **作用**：内存保护机制。绝对不能让行越界（ $< 0$ 或 $\ge$ 总行数），列同理。这行代码必须放在 `grid[nr][nc]` 判断的**最左边**，利用 C++ 的短路求值特性保护内存。

## 模块五：今日高频盲区与灵魂自测

**今日高频盲区总结**：

1. **动态数据的组装时机**：在排列组合问题中，只要底层数组的顺序/内容变了，依赖它算出来的整数（如 `val`）必须在循环内部**重新组装**。
    
2. **DFS 的底线意识**：分发物品的题目，**“人”或者“坑位”才是递归的深度限制**。必须把 `if(人 == 总数)` 写在函数的第一行作为终极拦截网，而不是用物品发没发完来拦截。
    

### 直击灵魂自测题（请遮挡下方答案自测）

**问题 1**：在遍历 `std::map<int, int> num` 时，如果我写 `for(auto aa : num)`，我想获取键和值，可以写成 `aa.first()` 和 `aa.second()` 吗？为什么？

**问题 2**：在二维网格中，总行数为 $n$，总列数为 $m$。我在验证坐标 $(x, y)$ 是否越界时，如果写出 `if(x >= 0 && x < m && y >= 0 && y < n)`，在什么样的网格长宽比下，代码会直接报段错误崩溃？

**问题 3**：对于分糖果那道题，如果我把主函数的入口写成 `dfs(0, 0, 1)`（代表从第 1 个人开始分），那么我 DFS 顶部的拦截条件应该是 `if(pepo == ?)` 才能正确停止并检查结果？

**【答案区】**

**答案 1**：绝对不行。`pair` 的 `first` 和 `second` 是属性（成员变量），不是函数（方法）。加括号 `()` 会被编译器当成函数调用从而报错。只能写 `aa.first`。

**答案 2**：在 $n > m$（行数大于列数，高瘦型网格）时会崩溃。因为 $x$ 代表行号，理论最大值可达 $n-1$。你错用 $m$ 去限制 $x$，当 $x \ge m$ 但其实还在合法行数内时，程序误判其越界；而 $y$ 代表列号，理论最大值为 $m-1$，你错用 $n$ 去限制，导致 $y$ 可以访问到 $\ge m$ 的非法内存，直接数组越界崩溃。

**答案 3**：应该是 `if(pepo == 8)`。因为你从 1 开始计数，分给第 1 个人、第 2 个人……当第 7 个人分完之后，程序进入下一层，此时 `pepo` 会变成 8。看到 8，说明前面 1-7 号小朋友都已经拿到糖果了，必须停手。
