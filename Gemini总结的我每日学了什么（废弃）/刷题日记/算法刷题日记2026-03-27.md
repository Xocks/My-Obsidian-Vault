### 模块一：今日知识清单与特定触发机制

1. **排序 + 一一配对（贪心算法）**
    
    - **Trigger (触发机制)**：当题目出现“两组相互独立的数据”、“任意两两配对/连接”、“求线段/差值/距离的最小或最大总和”时。第一反应：必须排序，从小到大一一对应消灭交叉。
        
2. **定长字符串扫描与提前截断（贪心匹配）**
    
    - **Trigger (触发机制)**：当题目要求“求出最多不重叠的特定子串”、“寻找某种特定字符组合”时。第一反应：从左到右扫描，一旦满足条件**立刻切割跳跃**，绝不把当前字符留给后续组合。
        
3. **无符号整数下溢出陷阱 ( `size_t` 漏洞)**
    
    - **Trigger (触发机制)**：只要在 C++ 的 `for` 或 `while` 循环条件中，看到 `容器.size() - 常数`。第一反应：极度危险！立刻思考如果容器大小小于该常数会怎样，必须改写为 `+ 常数 < 容器.size()` 或强制类型转换。
        
4. **快速排序的双指针划分 (Hoare Partition)**
    
    - **Trigger (触发机制)**：需要手写 O(N log N) 排序、找第 K 大的数、或者进行原地数组区间划分时。第一反应：选中间做基准值，左右指针 `do-while` 向中间逼近，不符则交换。
        
5. **置换环 (Permutation Cycles) **
    
    - **Trigger (触发机制)**：看到“长度为 N 的包含 1-N 的乱序排列”、“允许任意交换两点”、“求恢复升序的最少交换次数”。第一反应：不要想比较排序！直接套用图论闭环公式：`最少交换次数 = 元素总数 N - 环的个数 K`。

### 模块二：傻瓜式解题框架（思路提取）

以今天最颠覆直觉的**“置换环（书架还原）”**为例，建立通用思考路径：

- **第一步：空间初始化（极其关键）** 坚决弃用 `0` 索引。开辟大小为 `N + 1` 的数组 `a` 存数据，开辟大小为 `N + 1` 且全为 `false` 的布尔数组 `visited`。让书的编号和数组下标绝对对应。
    
- **第二步：核心遍历与顺藤摸瓜** 写一个外层 `for` 循环从 `1` 遍历到 `N`。只要遇到 `!visited[i]`，说明找到了一个新环，计数器 `cycles++`。 紧接着在内部写一个 `while (!visited[curr])`，核心跳转逻辑永远是：`visited[curr] = true; curr = a[curr];`。
    
- **第三步：结算输出** 完全不需要做任何真实的 `swap` 操作，直接输出 `N - cycles`。

### 模块三：我的源码级纠错与降维打击

#### 1. 原味纠错：最短距离 (遗漏核心逻辑)

C++

```
// 你的原代码
// ... 读取 com 和 btr ...
    
// 错误点：在计算两点距离之前，没有对无序的坐标进行排序。
// 纠正原因：如果不排序，你只是把输入的第 i 个显示器和第 i 个插座随机连在了一起。
// 物理上会产生大量“交叉连线”，导致距离总和远大于最优解。
// 必须在这里加上排序，让坐标一一对应：
sort(com.begin(), com.end());
sort(btr.begin(), btr.end());

long long res = 0;
for(int i = 0; i < n; ++i)
{
    res += abs(com[i] - btr[i]);
}
// ...
```

#### 2. 原味纠错：最多次数 (下溢出致命雷区)

C++

```
// 你的原代码
// ... 
int res = 0;
// 错误点：循环条件 i < str.size() - 2 会引发无符号整数下溢出。
// 纠正原因：str.size() 返回 size_t（无符号整数）。如果输入字符串长度只有 1，
// 1 - 2 在无符号整数域中会变成一个巨大的正数（比如 18446744073709551615）。
// 这会导致循环不仅不会跳出，还会疯狂访问未分配内存，直接段错误 (Segfault)。
// 纠正：移项改为加法。
for(int i = 0; i + 2 < str.size(); ++i) 
{
    if(tes(i))
    {
        i += 2;
        res++;
    }
}
// ...
```

#### 3. 降维打击：最多次数 (空间复杂度 O(N) -> O(1))

- **劣势**：你原本的 `cin >> str;` 会将整个长度可能达十万的字符串一次性吞入内存。
    
- **替代算法 (滑动窗口流式读取)**：不需要存整个字符串。准备一个大小为 3 的 `char window[3]`，利用 `while(cin >> c)` 一个字符一个字符地读。读一个，覆盖进窗口一个，只要窗口里凑齐了，就清空计数器。
    
- **降维优势**：无论测试用例输入十万、一千万还是十亿个字符，这个算法占用的内存永远只有 3 个字节（空间复杂度 O(1)），绝对不可能出现 Memory Limit Exceeded。

### 模块四：核心代码模版与保姆级拆解

为你提取 yxc 老师的**快速排序 (Hoare 划分)** 终极模板：

C++

```
void quick_sort(int q[], int l, int r)
{
    // 1. 递归终止条件
    if (l >= r) return;

    // 2. 初始化双指针与基准值
    int i = l - 1, j = r + 1, x = q[l + r >> 1];
    
    // 3. 核心区间划分 
    while (i < j)
    {
        do i ++ ; while (q[i] < x);
        do j -- ; while (q[j] > x);
        if (i < j) swap(q[i], q[j]);
    }
    
    // 4. 递归处理左右子区间
    quick_sort(q, l, j);
    quick_sort(q, j + 1, r);
}
```

**逐行剖析与参数透视：**

- **参数传递**：`q[]` 是数组，`l` 是当前操作区间的左边界索引，`r` 是右边界索引。返回值为空，因为是原地 (In-place) 排序。
    
- `if (l >= r) return;`：**边界保护**。如果区间内只有一个元素，或者给了一个非法的左大于右的区间，直接退出，防止无限递归。
    
- `i = l - 1, j = r + 1`：**目的**：为后面的 `do-while` 循环做准备。因为 `do-while` 是先无条件走一步，为了让第一步正好落在真实的 `l` 和 `r` 上，必须把指针初始位置放在区间外面。
    
- `x = q[l + r >> 1]`：**选取基准**。`>> 1` 是位运算，等同于除以 2。取中间值作为比较基准，防止在遇到近乎有序的数组时，快排退化成 O(N2)。
    
- `do i ++ ; while (q[i] < x);`：左指针向右跑，遇到比基准小的直接跳过，**目的**是找到第一个“本该在右边，却跑到了左边”的违规大数，然后停下。
    
- `if (i < j) swap(q[i], q[j]);`：只要两个指针没错过彼此，就把刚刚找到的两个违规的数互换位置。
    
- `quick_sort(q, l, j)`：**核心递归边界**。循环结束后，`j` 指针的位置就是左半区间（所有元素都 ≤x）的最后一个元素的索引。以此为界劈开。
