迭代器就是一个指针

### 为什么能加减整数/什么时候能/什么时候不能
一般数组的例如`arr[10]`就是`arr[arr.begin()+10]`所以能加减
#### 1. 原生指针：永远可以加减整数

在 C/C++ 中，原生指针（比如 `int* p`）加一个整数 `n`，表示在内存中向后移动 `n` 个元素的物理地址。因为原生指针指向的一定是一块连续的内存，所以这是绝对合法的。

#### 2. 迭代器阶级 1：“特权阶级”（随机访问迭代器）

**代表容器**：`vector`（动态数组）、`string`（字符串）、`deque`（双端队列）、原生数组。

这类容器的底层内存或者是完全连续的，或者是可以以 $O(1)$ 时间复杂度直接跳跃定位的。

因此，C++ 赋予了它们的迭代器**“随机访问”（Random Access）**的特权。它们在行为上长得和原生指针一模一样：

- ✅ 支持加减法：`it + n`、`it - n`
    
- ✅ 支持比大小：`it1 < it2`
    
- ✅ 支持下标访问：`it[n]`
    

所以你写的 `arr.erase(it, it + l1)` 对于 `vector` 来说是完全合法的。

#### 3. 迭代器阶级 2：“平民阶级”（双向迭代器 / 前向迭代器）

**代表容器**：`list`（双向链表）、`set`（红黑树集合）、`map`（红黑树字典）、`unordered_map`（哈希表）。

这些容器的底层**不是连续内存**，而是通过指针（节点）一个个连起来的。比如链表，你想要找第 5 个元素，电脑没办法直接算出来它的地址，只能顺着链表一步步爬过去。

因此，C++ **剥夺**了它们加减整数的权利，防止你误以为跳跃操作是 $O(1)$ 的：

- ❌ **不支持**加减法：`it + 5` （直接编译报错！）
    
- ❌ **不支持**比大小：`it1 < it2` （编译报错，只能用 `==` 或 `!=`）
    
- ✅ 只支持一步一步走：`++it` 或 `--it`
    

**如果你在比赛中用了 `list` 或 `set`，又非要往后移动 `n` 步怎么办？**

C++ 提供了专门的函数 `std::advance` 或 `std::next` 代替加号：

C++

```
#include <iostream>
#include <list>
#include <iterator> // advance 和 next 在这里

using namespace std;

int main() {
    list<int> myList = {10, 20, 30, 40, 50};
    auto it = myList.begin();

    // it = it + 2;  // 🔴 灾难！编译直接报错：没有匹配的 operator+

    // 正确写法 1：使用 advance (直接改变原迭代器)
    advance(it, 2); // 让 it 往后走 2 步，指向 30

    // 正确写法 2：使用 next (不改变原迭代器，返回一个新的)
    auto it2 = next(myList.begin(), 2); 

    return 0;
}
```

#### 总结

- 只要你用的是 **`vector`** 或 **`string`**，大胆地用 `it + n`，它们表现得和原生指针一模一样。
    
- 如果你用的是 **`list`、`set`、`map`**，千万别写 `it + n`，乖乖用 `++it` 循环，或者用 `std::advance(it, n)`。
    
- 在 STL 的设计哲学里：如果一个操作可能会非常慢（比如在链表里跳跃 $N$ 步是 $O(N)$ 的），C++ 就故意不给你重载 `+` 号，逼着你用特殊函数，从而在写代码时就能意识到这里的性能开销。

### vector和普通数组差异
#### 什么能用
**arr刚开始的内存定好了就无法改变**
普通数组没有`arr[i].begin()`等.但是有些情况能用`arr`替代`arr.begin()`
但像 `reverse()`、`sort()`、`lower_bound()` 这些函数，它们并不属于 `vector`，而是独立存在于 `<algorithm>` 头文件里的**通用算法**。
这些可以
```
// 对前 n 个元素排序 
sort(arr, arr + n); // 完全等价于 sort(v.begin(), v.end()) 

// 反转前 n 个元素 
reverse(arr, arr + n); // 完全等价于 reverse(v.begin(), v.end())
```
#### 插入（insert）和删除（erase）能用吗？（结构类）

**绝对不能用！**

这里要分清**“通用算法”**和**“容器专属招式”**的区别：

- `sort` 和 `reverse` 只是在已有的格子里互相交换数据，不改变格子的总数量。
    
- `insert` 和 `erase` 是要**改变内存长度**的。
    

`int arr[10]` 在声明的那一刻起，它在内存里就永远只有 10 个格子（静态分配）。你无法无中生有地硬挤进去第 11 个格子（不支持扩容），也无法把物理内存挖掉一块（不支持缩容）。 而 `vector` 内部写了极其复杂的代码，当你 `insert` 导致空间不够时，它会在后台偷偷找一块更大的新内存，把老数据搬过去，这就是所谓的“动态数组”。原生数组没有这个超能力。
