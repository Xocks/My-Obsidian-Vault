## 一、 核心概念与记忆画图

**人类思考方法（场景联想记忆）：**

在考场上如果忘了，立刻在脑子里想象**“在食堂排队买饭”**的画面：

- **队伍（Queue）：** 就是那个长条的通道。
    
- **入队（Push）：** 新来的人只能乖乖排在**队尾**。
    
- **出队（Pop）：** 打完饭的人从**队首**离开。
    
- **看一眼（Front）：** 食堂大妈抬头看一眼**现在轮到谁打饭了**（只看，不让人走）。

## 二、 queue 必背语法速查表

使用前别忘了引入头文件：`#include <queue>`。

|**代码写法**|**考场联想**|**作用说明**|**复杂度**|
|---|---|---|---|
|`q.push(x)`|**入队**（来人了）|将元素 `x` 放入队列的末尾。|$O(1)$|
|`q.front()`|**看队首**（轮到谁了）|返回队列最前面那个元素的值。（**注意：不删除该元素！**）|$O(1)$|
|`q.pop()`|**出队**（打完饭走人）|删除队列最前面的那个元素。（**注意：它没有返回值！**不能写 `cout << q.pop()`）|$O(1)$|
|`q.empty()`|**判空**（队伍空了吗）|检查队列是否为空。空返回 `true`，非空返回 `false`。|$O(1)$|
|`q.size()`|**查人数**（有多长）|返回队列中当前有多少个元素。|$O(1)$|
|`q.back()`|**看队尾**（最后是谁）|返回排在队列最后面的那个元素的值。（考试中使用频率极低）|$O(1)$|

> **⚠️ 考场致命易错点（你上次遇到的坑）：**
> 
> 永远、永远不要对一个**空队列**执行 `q.front()` 或 `q.pop()`！这会导致未定义行为，通常表现为程序崩溃（RE）。
> 
> **肌肉记忆：** 只要你的手敲出了 `.front()` 或 `.pop()`，眼睛立刻往上一行扫，看看有没有写 `if (!q.empty())` 或 `while (!q.empty())`。

## 三、 标准语法测试模版

这里给你一个带注释的完整模版，把上面的语法串联起来。你可以把它当成一个“黑盒”，看看数据进去和出来的样子。

C++

```
#include <iostream>
#include <queue>

using namespace std;

int main() {
    // 1. 创建一个装整数的队列
    queue<int> q;

    // 2. 入队操作 (排队买饭)
    q.push(10); // 10 排在第一
    q.push(20); // 20 排在第二
    q.push(30); // 30 排在第三
    
    // 此时队列长这样： (队首) 10 <- 20 <- 30 (队尾)

    // 3. 查看队列基本信息
    cout << "当前排队人数: " << q.size() << "\n"; // 输出 3

    // 4. 标准的“清空并打印队列”的写法 (考场最常用写法)
    // 只要队列不为空，就一直处理
    while (!q.empty()) {
        // 第一步：先看看现在轮到谁了
        int current = q.front(); 
        cout << "正在处理: " << current << "\n";
        
        // 第二步：处理完了，让他出队走人
        q.pop(); 
    }

    // 5. 再次检查判空
    if (q.empty()) {
        cout << "所有人都打完饭了，队伍空了！\n";
    }

    return 0;
}
```
