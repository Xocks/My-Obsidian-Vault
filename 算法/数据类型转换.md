### 浮点和整数
**你的代码：**
```
double one; cin >> one;
long long l = one*100; // ❌ 隐藏的杀手！
```
**为什么错：** 计算机底层存浮点数（小数）用的是二进制，很多十进制小数是无法精确表示的。 比如，当你输入 `14.10` 时，由于计算机精度问题，它在内存里存的可能是 `14.09999999999999`。 当你执行 `one * 100` 时，得到的是 `1409.9999999999`。 当你把它赋给 `long long` 时，C++ 的规矩是**直接抹掉小数点后所有的数字（向下取整）**！于是 `1409.9999...` 就变成了 `1409`（也就是 14.09 元），你活生生亏了 1 分钱！

**正解：使用四舍五入 `round()`** 既然我们知道它是极其接近真实数字的，我们只要加上 `<cmath>` 库里的 `round()` 函数，让它四舍五入，就能完美解决： `long long l = round(one * 100);` （这样 1409.999 就会被拯救回 1410）。
