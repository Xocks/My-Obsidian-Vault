### 一、 贪心题目的四大核心门派（分类）

做贪心题，第一步就是“认门派”。几乎所有初中级贪心题，都可以归入以下四类：

#### 1. 区间类贪心（绝对的主力，面试/竞赛必考）

- **长什么样：** 给你一堆带“开始时间”和“结束时间”的线段（活动、会议、区间）。
    
- **经典提问：** “最多能参加几个不冲突的活动？”、“最少需要几个房间能安排下所有会议？”、“最少几支箭能射穿所有重叠的气球？”
    
- **核心策略：** 按照右端点（结束时间）从小到大排序，或者按左端点排序。
    

#### 2. 分配与匹配类贪心（双指针辅助）

- **长什么样：** 给你两组数据，让你把它们两两配对，达到某种极值。
    
- **经典提问：** “田忌赛马（怎么出战赢最多）”、“分发饼干（大小不一的饼干喂给胃口不一的孩子，最多喂饱几个）”、“救生艇（两人一船，有重量上限，最少要几艘船）”。
    
- **核心策略：** 把两组数据分别从小到大排序。用两个指针，大胃口配大饼干（或者小配小），一旦满足就齐步往前走。
    

#### 3. 序列操作类贪心（就地修改）

- **长什么样：** 给你一个数组或字符串，每次只能按特定规则修改相邻元素。
    
- **经典提问：** 就是你刚刚做过的“翻硬币”，或者“最少操作几次让数组变成全零”。
    
- **核心策略：** 找到一个“必须修改的强制推导点”（通常是最左边或最右边），从一端推到另一端，不回头。
    

#### 4. 堆贪心（优先队列辅助，进阶）

- **长什么样：** 数据是动态变化的，你每次都要从当前所有可用选项里挑一个最大的或最小的。
    
- **经典提问：** “合并果子（每次选两堆最小的合并，总体力消耗最小）”、“安排任务（每次选剩余次数最多的任务先做）”。
    
- **核心策略：** 不用传统的 `sort`，而是把数据扔进大根堆/小根堆（C++ 里的 `priority_queue`），每次拿堆顶元素操作。

### 二、 极简思考步骤（大脑推演逻辑）

拿到一道疑似贪心的题，按照这三步在脑子里过一遍：

1. **直觉锁定（找局部最优）：**
    
    看到题目问“最多”、“最少”，直觉去想：“如果在当前这一步，我怎么做最占便宜？”
    
    _比如：排队打水，怎么让所有人总等待时间最短？直觉：让打水最快的人先上！_
    
2. **灵魂拷问（举反例推翻）：**
    
    立马在脑子里试图找茬：“如果我一直占眼前的便宜，会不会导致后面全盘崩溃？”
    
    _比如：刚才打水的例子，如果让打水慢的人先上，他会拖累后面所有人，等待时间之和绝对更大。反例不成立，说明直觉是对的。_
    
3. **敲定规则（转化为排序）：**
    
    只要找不到反例，就大胆确认贪心策略。把你的“直觉”翻译成 C++ 的 `cmp` 排序规则。

### 三、 怎么写？（万能代码框架）

对于前两类最常见的贪心（区间类、分配类），它们的代码结构惊人地一致，核心就是 **结构体 + 自定义排序 + 单向遍历**。

**明显的替代算法差异：** 如果你在“灵魂拷问”阶段找到了反例，发现“眼前的最优会破坏未来的最优”（也就是存在**后效性**，比如背包问题中，拿了眼前性价比最高的物品，导致背包装不下后来更值钱的物品），贪心就彻底失效了。此时必须换用**动态规划（DP）**。贪心的时间复杂度通常是极快的 $O(N \log N)$（全花在排序上），而 DP 往往需要 $O(N^2)$ 或更高的时空复杂度来记录所有可能的状态。

下面是为你提取的通用模板，请仔细看注释里的“人类思考位置”：

C++

```
#include <bits/stdc++.h>
using namespace std;

// 第一步：定义数据载体
// 把题目中绑定的数据打包在一起（比如活动的开始和结束时间，人的身高和体重）
struct Item {
    int start;
    int end;
    // 可以有更多属性...
};

// 第二步：人类智慧的结晶（定义贪心规则）
// 核心逻辑全在这个函数里！告诉计算机谁排前面，谁排后面。
bool cmp(Item a, Item b) {
    // 【举例策略】：按结束时间早的排前面
    // 思考：谁结束得早，留给后面的可用时间就越多，我能安排的活动总数才可能最多。
    if (a.end != b.end) {
        return a.end < b.end; 
    }
    // 如果结束时间一样，可以附加第二规则（视具体题目而定，通常不需要）
    return a.start < b.start; 
}

int main() {
    int n;
    cin >> n;
    vector<Item> arr(n);
    for (int i = 0; i < n; i++) {
        cin >> arr[i].start >> arr[i].end;
    }

    // 第三步：无脑排序
    // 这一步过后，数组已经按照你的“贪婪欲望”排好队了
    sort(arr.begin(), arr.end(), cmp);

    // 第四步：单向遍历，收割结果
    int res = 0;       // 记录最终的最优解
    int current = -1;  // 记录当前的环境状态（比如：上一个活动几点结束的）

    for (int i = 0; i < n; i++) {
        // 如果当前这个物品（活动），满足客观条件（开始时间晚于上一个结束时间）
        if (arr[i].start >= current) {
            // 毫不犹豫地拿下它！
            res++; 
            // 关键：更新当前的环境状态，为下一个做准备
            current = arr[i].end; 
        }
    }

    // 第五步：输出结果
    cout << res << endl;

    return 0;
}
```

从结构可以看出，贪心代码极度精简，**难点永远在 `cmp` 函数的构思上**，也就是你找的“贪心标准”对不对。
