# 🏆 C++ 算法竞赛核心武器库：数组与 STL 终极速查笔记

在算法竞赛（如蓝桥杯、ACM/ICPC）中，熟练掌握 STL 能够极大缩短编码时间、降低 Bug 率。本笔记聚焦于 **“用最少的代码行数、最优的时间复杂度完成数组的增删改查”**。

> 💡 **核心心法：管家与外包团队**
> 
> - **`vector` 内部函数（大堂经理）：** 拥有最高权限，负责修改物理容量（扩容、释放内存）。如 `push_back`, `erase`, `reserve`。
>     
> - **`<algorithm>` 算法库（外包团队）：** 认准迭代器（放大镜）干活，**绝对无法**改变数组的物理长度。如 `sort`, `lower_bound`, `unique`。
>     

---

## 🚀 一、 增 (Create & Insert)

尽量避免在数组中间写 `for` 循环单点插入，多利用尾部追加和区间/批量操作。

|**操作类型**|**推荐语法**|**时间复杂度**|**竞赛场景说明**|
|---|---|---|---|
|**尾部追加**|`v.push_back(x)`|均摊 $O(1)$|最常用的读入方式。|
|**高效追加**|`v.emplace_back(args)`|均摊 $O(1)$|存结构体时极速构造，省去拷贝（如 `v.emplace_back(x, y)`）。|
|**批量插入**|`v.insert(it, first, last)`|$O(N+M)$|将一个数组直接塞入 `vector` 中间，底层只平移一次元素。|
|**递增填充**|`iota(v.begin(), v.end(), 0)`|$O(N)$|神技！瞬间生成 `0, 1, 2... n-1` 的序列，常用于离散化或索引排序。|

C++

```
// 批量插入示例：在 v[2] 的位置，插入 arr 的所有元素
vector<int> v = {1, 2, 8, 9};
int arr[] = {3, 4, 5};
v.insert(v.begin() + 2, arr, arr + 3); // {1, 2, 3, 4, 5, 8, 9}
```

## ✂️ 二、 删 (Delete)

删除操作会导致后续元素左移，切忌在遍历时随意单点删除。

|**操作类型**|**推荐语法**|**时间复杂度**|**竞赛场景说明**|
|---|---|---|---|
|**尾部弹出**|`v.pop_back()`|$O(1)$|常用于模拟栈结构 (DFS 状态回溯)。|
|**区间删除**|`v.erase(first, last)`|$O(N)$|配合算法库服用，实现真正的物理空间清理。|
|**彻底清空**|`v.clear()`|$O(N)$|清空所有元素，但**不会**释放申请过的物理内存 (Capacity)。|

## 🛠 三、 改与重置 (Modify & Initialize)

处理多组测试用例 (Multiple Test Cases) 时，快速、干净地重置状态是刚需。

|**操作类型**|**推荐语法**|**时间复杂度**|**竞赛场景说明**|
|---|---|---|---|
|**批量重置**|`fill(first, last, val)`|$O(N)$|极速将区间内的元素全变成 `-1` 或 `0`，替代 `for` 循环。|
|**推翻重构**|`v.assign(n, val)`|$O(N)$|销毁原数据，直接用 $n$ 个 `val` 覆盖当前 `vector`。|
|**空间预留**|`v.reserve(n)`|$O(N)$ 分配|**防 TLE 核心！** 提前向系统申请 $n$ 个物理空间，防止反复搬家。|

## 🔍 四、 查与统计 (Search & Count)

查找是逻辑构建的核心。绝不要手写 `for` 循环找最大值或统计个数！

### 1. 线性查找 (Linear Search)

- **找单个值:** `auto it = find(v.begin(), v.end(), val)`
    
- **条件查找 (神技):** `auto it = find_if(v.begin(), v.end(), 限制条件)`
    
    - _返回满足 Lambda 表达式条件的第一个元素的迭代器。_
        
- **找连续子数组:** `auto it = search(主.begin(), 主.end(), 子.begin(), 子.end())`
    
- **统计次数:** `int cnt = count(v.begin(), v.end(), val)`
    
- **找极值:** `int max_val = *max_element(v.begin(), v.end())`
    
    - _注意：`max_element` 返回迭代器，必须加 `*` 解引用，求下标用 `- v.begin()`。_
        

### 2. 二分查找 (Binary Search - 必完全掌握)

**⚠️ 前提：数组必须已排序！时间复杂度均为 $O(\log N)$。**

|**函数名**|**核心功能**|**实战代码示例**|
|---|---|---|
|**`lower_bound`**|找第一个 **$\ge$** 目标值的位置|`int idx = lower_bound(v.begin(), v.end(), 3) - v.begin();`|
|**`upper_bound`**|找第一个 **$>$** 目标值的位置|`int idx = upper_bound(v.begin(), v.end(), 3) - v.begin();`|
|**`binary_search`**|判断元素 **存不存在** (返回 bool)|`if (binary_search(v.begin(), v.end(), 3))`|

## ✨ 五、 竞赛黑科技组合连招 (Advanced Tricks)

能用一行代码解决的，绝不写十行。以下是拉开代码量差距的关键工具：

### 1. 完美去重 (Deduplication)

由于 `unique`（外包团队）只能去除相邻重复且无权改动物理长度，必须配合 `sort` 和 `erase`（大堂经理）。

C++

```
// 排序 -> 移动重复项 -> 物理切除废弃尾巴
sort(v.begin(), v.end());
v.erase(unique(v.begin(), v.end()), v.end());
```

### 2. $O(N)$ 快速选择 (nth_element)

只需求数组第 $K$ 小的数，但不想花 $O(N \log N)$ 排序？

C++

```
// 把第 3 小的数放在 v[2] 的位置，左边都比它小，右边都比它大 (不完全有序)
nth_element(v.begin(), v.begin() + 2, v.end());
```

### 3. 字典序全排列 (Next Permutation)

暴力杯拿分利器，彻底替代手写全排列 DFS。

C++

```
vector<int> v = {1, 2, 3}; // 必须从最小字典序开始
do {
    // 依次输出 123, 132, 213, 231...
} while (next_permutation(v.begin(), v.end()));
```

### 4. 物理翻转与旋转

- **翻转:** `reverse(v.begin(), v.end());` (常用于高精度倒序)
    
- **旋转:** `rotate(v.begin(), v.begin() + 2, v.end());` (将前两个元素平移到末尾)

## 💣 六、 致命排雷指南

1. **传参引发的 TLE/MLE：**
    
    在 DFS 或普通函数中传递 `vector`，**必须使用引用 `&`**！
    
    - ❌ 错：`void dfs(vector<int> v)` （每次递归复制几十万数据，瞬间爆炸）
        
    - ✅ 对：`void dfs(vector<int>& v)` （传递引用地址，速度 $O(1)$）
        
2. **`size()` 的无符号陷阱：**
    
    `v.size()` 返回的是无符号整数（`size_t`）。当数组为空时，`v.size() - 1` 会下溢出一个极大的正数，导致数组越界。
    
    - **习惯写法：** `(int)v.size() - 1` 或者 `!v.empty()`。
        
3. **栈溢出 (Stack Overflow)：**
    
    如果题目数据量 $N \ge 10^5$，**绝对不要**在 `main` 函数里开 `int arr[100000]`，必须开在全局作用域。如果用 `vector` 则无所谓（数据在堆区）。
