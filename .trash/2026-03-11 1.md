# 算法刷题日记 - 2026-03-11

## 核心知识点概览

今日主要攻克了**深度优先搜索（DFS）的枚举模型**、**字符串处理**、**过程模拟**以及**状态机与空间优化**。通过手写分析，彻底理清了在递归中“何时用 for 循环”与“何时不用 for 循环”的底层逻辑。

## 题目回顾与核心总结

### 1. DFS 基础枚举系列

- **[B3622] 枚举子集 (指数型枚举)**
    
    - **核心思维**：“选或不选”模型（关注物品本身）。
        
    - **实现方式**：不需要 for 循环。在一层递归中写出两个平行的搜索分支（选 / 不选），深度走到叶子节点即输出。时间复杂度 $O(2^n)$。
        
- **[B3623] 枚举排列 (排列型枚举)**
    
    - **核心思维**：“位置找人”模型（关注坑位）。
        
    - **实现方式**：必须使用 for 循环。在当前位置用 for 循环扫视所有候选人，配合 `bool used[]` 数组记录状态（占用与回溯释放）。时间复杂度 $O(n!)$。
        
- **[P10448] 组合型枚举**
    
    - **优化思维**：组合不讲究顺序，为了避免重复，引入 `num` 参数强制后续选的数必须大于前面的数，实现严格递增。
        
    - **剪枝技巧 (Pruning)**：在递归开头加入判断，如果“剩余可选的数字数量”小于“还缺少的坑位数量”，直接 `return` 砍掉废树枝，大幅提升效率。
        

### 2. 模拟与状态机系列

- **[P8722] 日期识别**
    
    - **核心思维**：字符串提取与转换。
        
    - **实现技巧**：利用 `map<string, int>` 实现英文月份到数字的哈希映射。利用 ASCII 码偏移量（`char - '0'`）将字符转换为真实整数。切记 C++ 中单引号 `' '` 用于字符，双引号 `" "` 用于字符串。
        
- **[P12170] 攻击次数 (蓝桥杯过程模拟)**
    
    - **核心思维**：时间步进（Time Stepping）。使用 `while(hp > 0)` 配合 `回合数++` 进行直观的按步模拟。
        
    - **进阶拓展**：当数据量极大导致超时，可寻找各攻击周期的**最小公倍数 (LCM)**，将 $O(N)$ 复杂度的逐个模拟转化为 $O(1)$ 复杂度的周期除法与取余运算。
        
- **[P10899] 劲舞团 (蓝桥杯状态机)**
    
    - **核心思维**：If-Else 互斥原则与状态流转。
        
        1. **冷启动/断连重置**：敲对时，若连击为 0，赋值为 1。
            
        2. **连击判定**：敲对且处于连击状态时，判断 `curr_ms - last_ms <= 1000`。
            
        3. **清零**：一旦敲错，连击数必须立刻归零。
            
    - **空间极致优化 (滚动变量)**：抛弃了原本 $O(N)$ 的十万级大数组。由于每次计算只需比对“上一次的时间”，因此只保留 `curr_ms` 和 `last_ms` 两个变量互相交接班，成功将空间复杂度降为 $O(1)$。
        

## 珍藏模版：DFS 选与不选（组合型+剪枝）

C++

```
void dfs_pick(int st, int num) {
    // 递归出口 1：选够了 k 个数
    if (st == k) {
        // 输出并 return
        return;
    }
    // 递归出口 2：越界
    if (num > n) return;
    
    // 【终极剪枝】：剩下的数不够填满剩下的坑位，直接回头！
    if (n - num + 1 < k - st) return;

    // 决定 1：选当前数字 num
    arr[st] = num;
    dfs_pick(st + 1, num + 1);

    // 决定 2：不选当前数字 num（坑位不变，候选数字换下一个）
    dfs_pick(st, num + 1);
}
```
